# 验收、变更与排障

本文档只保留当前有效的本地验收与排障方式。

## 1. 推荐验收顺序

先看运行摘要：

```bash
rtk python D:/Project/fms/scripts/local_runtime_manager.py summary
rtk python D:/Project/fms/scripts/local_runtime_manager.py check --probes
```

再看依赖服务：

```bash
rtk python D:/Project/fms/scripts/start_local_services.py --print-plan
```

如需 Kafka / CDC：

```bash
rtk python D:/Project/fms/scripts/bootstrap_local_kafka_debezium.py
```

如只需确保 Kafka 栈可用：

```bash
rtk python D:/Project/fms/scripts/bootstrap_local_kafka_debezium.py --startup-only
```

## 2. Kafka 验收标准

启动成功后，至少应满足：

- `zookeeper` 已就绪
- `kafka` 已就绪
- `kafka-init` 已成功退出
- `kafka-connect` 已健康
- `debezium-init` 已成功退出
- Kafka Connect 中 `oms-debezium-connector` 和 `crm-debezium-connector` 都在运行

推荐检查：

```bash
rtk python -m pytest D:/Project/fms/tests/test_local_kafka_compose.py -q
rtk python -m pytest D:/Project/fms/tests/test_governance_status_services.py::test_kafka_cluster_status_exposes_local_kafka_and_debezium -q
rtk python -m pytest D:/Project/fms/tests/test_api_integration.py::test_kafka_cluster_status_endpoint -q
```

## 3. 变更边界

Kafka 相关改动现在必须遵守：

- 只保留 `docker-compose.local-kafka.yml` 这一套本地 Kafka Compose
- 不要把 Kafka 服务重新加回 `docker-compose.yml`
- 不要改掉 `kafka:29092` 和 `kafka-connect:8083` 这两个业务可见地址
- 不要删除 `pms-network` 上的别名桥接

## 4. 常见问题

### 4.1 只启动 Kafka 栈时报共享网络不存在

当前标准入口已经会先确保共享网络存在：

```bash
rtk python D:/Project/fms/scripts/bootstrap_local_kafka_debezium.py --startup-only
```

不要再手工依赖“必须先起 app 才有 `pms-network`”这种隐含前提。

### 4.2 旧 Kafka 容器和卷还留在宿主机

先 dry-run：

```bash
rtk python D:/Project/fms/scripts/cleanup_local_runtime.py
```

确认后再执行：

```bash
rtk python D:/Project/fms/scripts/cleanup_local_runtime.py --apply
```

这个脚本只会清理已经废弃的旧 Kafka Docker 资源，不会自动卸载 Python、Docker Desktop、WSL、Node.js 等通用宿主机软件。

### 4.3 app 访问不到 Kafka

优先排查：

1. Kafka 栈是否通过标准入口启动
2. `pms-local-kafka` 是否已接入 `pms-network`
3. `kafka` 别名是否仍然存在

必要时查看：

```bash
rtk docker network inspect pms-network
rtk docker ps
```

### 4.4 Kafka Connect 启动了，但 Debezium connector 没注册

先看：

```bash
rtk docker logs pms-local-kafka-connect
rtk docker logs pms-local-debezium-init
```

然后重新执行：

```bash
rtk python D:/Project/fms/scripts/bootstrap_local_kafka_debezium.py --startup-only
```

### 4.5 CDC 样例验收失败

使用完整验收模式重跑：

```bash
rtk python D:/Project/fms/scripts/bootstrap_local_kafka_debezium.py
```

它会重新执行：

- 源表准备
- Debezium 初始化
- 样例数据写入
- CDC 消息校验
