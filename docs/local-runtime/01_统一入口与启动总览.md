# 统一入口与启动总览

本地环境现在按固定顺序启动，不再混用手工命令、历史脚本和多套 Kafka 方案。

## 1. 唯一入口

- Python 依赖安装：`python D:/Project/fms/scripts/install_python_deps.py --run-check`
- 宿主机软件检查：`python D:/Project/fms/scripts/install_local_software.py`
- 依赖服务启动：`python D:/Project/fms/scripts/start_local_services.py`
- 废弃 Docker 资源清理：`python D:/Project/fms/scripts/cleanup_local_runtime.py`
- 应用本机启动：`powershell -NoProfile -File D:/Project/fms/scripts/start_local.ps1`
- 运行摘要与检查：`python D:/Project/fms/scripts/local_runtime_manager.py summary`

## 2. 日常启动顺序

首次准备：

```bash
python D:/Project/fms/scripts/install_python_deps.py --run-check
python D:/Project/fms/scripts/install_local_software.py
```

启动依赖服务：

```bash
python D:/Project/fms/scripts/start_local_services.py
python D:/Project/fms/scripts/start_local_services.py --with-kafka
python D:/Project/fms/scripts/start_local_services.py --with-platform
python D:/Project/fms/scripts/start_local_services.py --with-ollama
```

启动本机 backend：

```bash
powershell -NoProfile -File D:/Project/fms/scripts/start_local.ps1
```

等价后端命令：

```bash
python D:/Project/fms/scripts/local_runtime_manager.py up --skip-deps
```

## 3. 重资源落盘规则

- 统一规则：本地拉取型资源、模型缓存、工具软件都放在 `D:\aitools`，不要放在 `C:`。
- 当前默认目录：
  - `LOCAL_OLLAMA_DATA_DIR=D:/aitools/fms/ollama`
  - `LOCAL_MODEL_CACHE_DIR=D:/aitools/fms/model-cache`
- 首次准备建议直接执行：

```bash
python D:/Project/fms/scripts/install_local_software.py --apply
```

如果你还要求 Docker 镜像层和 WSL 磁盘也完全不落到 `C:`，需要额外把 Docker Desktop 磁盘镜像与 WSL 发行版迁移到 `D:\aitools`；Compose 只能控制绑定挂载目录。

## 4. Kafka 唯一启动方式

标准方式：

```bash
python D:/Project/fms/scripts/start_local_services.py --with-kafka
```

高级方式：

```bash
python D:/Project/fms/scripts/bootstrap_local_kafka_debezium.py --startup-only
```

说明：

- 启动脚本会先确保共享网络存在，再用 `docker-compose.local-kafka.yml` 拉起 Kafka 栈。
- 这套 Kafka 栈会自动完成 Debezium 初始化，不需要再从 `docker-compose.yml` 启动重复服务。
- `app` 继续使用 `kafka:29092`，不需要改 `.env` 或业务代码。

## 5. 关键端口

- Kong Proxy：`8000`
- Kong Admin：`8001`
- 应用直连：`18000`
- Kafka 外部访问：`9092`
- Kafka Connect 外部访问：`8083`
- OpenSearch：`19200`
- Neo4j Bolt：`17687`
- Keycloak OIDC：`18082`
- Flink JobManager：`18081`

## 6. 关键边界

- `docker-compose.yml` 只保留基础数据层和可选业务镜像服务，不再承载 Kafka 栈，也不再作为默认 backend 启动入口。
- `docker-compose.local-kafka.yml` 是唯一 Kafka / CDC Compose。
- 不要重新引入第二套 Kafka Compose 或第二套容器命名。
- 本地 AI 大资源通过 Compose 绑定挂载统一落到 `D:\aitools`。
