# 本地验收方案

## 1. 目标

本文档用于指导当前项目的本地整体验收，目标是：

- 覆盖原始需求的核心业务闭环
- 验证主链路与异常链路不断链
- 验证治理、巡检、交付快照与页面展示一致
- 为交付沟通、阶段验收和用户手册编写提供统一依据

当前项目的本地交付口径以如下结论为准：

- 交付巡检状态：`ready`
- 可进入交付/验收沟通
- 已验证范围：本地业务闭环 + 治理验收
- 未纳入口径：外部 staging、真实第三方凭证全量联调、多环境生产部署

## 2. 验收范围

### 2.1 原始业务主链路

覆盖以下核心闭环：

1. 选品任务创建
2. 选品分析执行
3. 多级审批流
4. 采纳推荐
5. SCM / WMS / OMS 执行编排
6. CRM / FMS / BI 回流
7. 执行后再评分
8. 利润飞轮状态

### 2.2 异常路径

覆盖以下关键异常分支：

1. 审批拒绝闭环
2. 非 GO 任务禁止采纳
3. execution_result 缺失时安全退出
4. 部分反馈系统未 ready 时正确暴露 gap

### 2.3 治理与交付巡检

覆盖以下治理与交付资产：

1. business config governance
2. rag governance
3. operations governance overview
4. delivery readiness snapshot
5. delivery conclusion summary
6. delivery scope boundary

### 2.4 页面与展示

覆盖以下现有页面：

1. 运营台 `/operations`
2. 知识库工作台 `/knowledge`
3. 选品工作台 `/workbench/selection`
4. 报告中心 `/reports`
5. Agent 平台 `/agents`

## 3. 验收前置条件

### 3.1 本地基础依赖

建议至少保证以下依赖可用：

- Python 运行环境
- PostgreSQL / SQLite fallback
- Redis
- Qdrant
- Kafka
- OpenSearch（若需要关键词检索）
- Ollama（若需要本地模型链路）

### 3.2 建议先做健康检查

先确认基础状态接口：

- `/health`
- `/ready`
- `/api/v1/info`
- `/api/v1/system/delivery-readiness`
- `/api/v1/system/delivery-conclusion`
- `/api/v1/system/delivery-scope-boundary`

## 4. 推荐执行顺序

### 步骤 1：主链正常路径验收

执行：

```bash
rtk python "scripts/run_local_selection_main_chain_acceptance.py"
```

通过标准：

- 任务创建成功
- 本地执行完成
- 多级审批通过
- 人工干预记录成功
- 采纳推荐成功
- 最终任务详情一致

### 步骤 2：主链闭环验收

执行：

```bash
rtk python "scripts/run_local_selection_close_loop_acceptance.py"
```

通过标准：

- adoption execution completed
- execution feedback synced
- rescore persisted to task
- feature asset ready
- feedback loop status ready
- profit trace ready
- audit logs captured

### 步骤 3：主链异常路径验收

执行：

```bash
rtk python "scripts/run_local_main_chain_exception_acceptance.py"
```

通过标准：

- approval_reject_closed
- non_go_adoption_blocked
- missing_execution_result_safe_exit
- partial_feedback_loop_exposes_gap

### 步骤 4：配置治理验收

执行：

```bash
rtk python "scripts/run_business_config_governance_runbook.py"
```

通过标准：

- export 正常
- acceptance 正常
- rollback 正常
- runbook 结果 `ok=true`

### 步骤 5：RAG 治理验收

执行：

```bash
rtk python "scripts/run_rag_governance_runbook.py"
```

通过标准：

- feedback learning 正常
- evaluation 正常
- dashboard 正常
- runbook 结果 `ok=true`

### 步骤 6：交付巡检快照

执行：

```bash
rtk python "scripts/run_delivery_readiness_snapshot.py"
```

通过标准：

- business_governance_ok = true
- rag_governance_ok = true
- main_chain_exceptions_ok = true
- operations_overview_status = ready
- artifact_ready_count = artifact_total_count

### 步骤 7：交付结论与边界说明

执行：

```bash
rtk python "scripts/export_delivery_conclusion_summary.py"
rtk python "scripts/export_delivery_scope_boundary.py"
```

通过标准：

- 交付结论为 ready / 可进入交付沟通
- 已验证边界与未纳入口径的范围明确

## 5. 页面验收建议

### 5.1 运营台 `/operations`

检查项：

- 交付巡检状态
- 主链异常状态
- 配置治理状态
- RAG 治理状态
- 交付结论
- 导出交付巡检快照按钮
- GPU 监控状态

### 5.2 知识库工作台 `/knowledge`

检查项：

- 默认评测条数
- feedback 样本数
- 合并基线数
- 覆盖率
- 最近更新时间
- RAG 评测 artifact 路径

## 6. 核心产物

执行完本地验收后，至少应生成如下关键产物：

- `artifacts/local_main_chain/.../summary.json`
- `artifacts/local_close_loop/.../summary.json`
- `artifacts/local_main_chain_exceptions/.../summary.json`
- `artifacts/ops/business_config_governance_runbook.json`
- `artifacts/knowledge/rag_governance_runbook.json`
- `artifacts/ops/delivery_readiness_snapshot.json`
- `artifacts/ops/delivery_conclusion_summary.json`
- `artifacts/ops/delivery_scope_boundary.json`

## 7. 验收结论口径

若以上步骤均通过，则当前本地验收结论可表述为：

- 原始业务主链本地闭环已通过
- 主链关键异常路径本地验收已通过
- 配置治理与 RAG 治理链路已通过
- 当前已具备本地交付与验收沟通条件

## 8. 当前边界说明

当前不应对外误报为“已完成”的范围：

- 外部 staging 环境联调
- 真实第三方凭证全量联调
- 超出当前本地闭环口径的生产级多环境部署
- 任务清单中明确标记为外部阻塞/高配资源依赖的事项
