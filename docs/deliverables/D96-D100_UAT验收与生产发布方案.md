## 2. UAT验收方案

### 2.1 当前推荐验收环境

> 以下口径以**当前仓库真实可执行基线**为准，不再默认按“完整生产多服务集群”描述。

| 环境层级 | 当前建议 | 说明 |
|------|------|------|
| 本地演示/UAT基线 | 前端本地 + Playwright mock API + 后端最小真实接口校验 | 用于快速演示工作台、报告、运营、利润中枢、审计/治理能力 |
| 本地接口验收 | FastAPI 本地运行 + pytest 集成用例 | 用于安全状态、LLM 拦截、审计查询、指标看板等真实接口验收 |
| 外部环境验收 | 仅对 Triton / Kong / ERP 联调项启用 | 当前仍属外部环境阻塞项，不纳入本地 UAT 通过口径 |

### 2.3 当前不纳入本地 UAT 通过口径的阻塞项

| 类别 | 阻塞项 | 原因 |
|------|------|------|
| AI 中台 | Triton embedding/rerank/多模态真实链路 | 缺真实 Triton 服务与模型仓 |
| 网关 | Kong test/preprod/prod 真实接入 | 缺远端环境与探测位 |
| 业务集成 | WMS / SCM / FMS / OMS 真实闭环 | 缺企业联调窗口和真实样本 |
| 企业安全 | SSO / IdP / WAF 联动 | 缺企业安全环境 |

### 2.4 当前 UAT 通过标准

- P0 本地高价值项通过条件：
  1. 对应页面/接口/脚本可重复执行
  2. 存在截图、报告、JSON 工件或测试结果作为证据
  3. 至少 1 条自动化回归通过
- 本地 UAT 不等于 enterprise-pass：
  - 涉及 Triton、Kong、ERP、SSO 的项必须待外部环境补验
- 签字结论建议分三档：
  - **通过（本地基线）**
  - **有条件通过（待外部环境补验）**
  - **不通过**

## 5. 回滚方案

### 5.1 回滚触发条件

| 条件 | 阈值 | 触发动作 |
|------|------|---------|
| 错误率 | >5% | 自动回滚 |
| P99延迟 | >500ms | 人工评估 |
| CPU使用率 | >90% | 人工评估 |
| 内存使用率 | >95% | 人工评估 |
| 业务异常 | 关键功能不可用 | 立即回滚 |

### 5.2 回滚脚本

```bash
#!/bin/bash
set -e

PREVIOUS_VERSION=${1:-"v0.9.0"}
NAMESPACE="fms-production"

echo "=== 开始回滚流程 ==="
echo "回滚到版本: $PREVIOUS_VERSION"

# Step 1: 停止流量
echo "Step 1: 停止新版本流量..."
kubectl patch virtualservice pms-api -n $NAMESPACE \
    --type merge -p '{"spec":{"http":[{"route":[{"destination":{"host":"pms-api","subset":"stable"},"weight":100}]}]}}'

# Step 2: 回滚应用
echo "Step 2: 回滚应用..."
helm rollback fms -n $NAMESPACE

# Step 3: 回滚数据库
echo "Step 3: 回滚数据库..."
kubectl exec -n fms-data deployment/postgres -- \
    psql -c "SELECT pg_restore('/backup/pre_release_${PREVIOUS_VERSION}.sql');"

# Step 4: 验证回滚
echo "Step 4: 验证回滚..."
kubectl rollout status deployment/pms-api -n $NAMESPACE

# Step 5: 恢复流量
echo "Step 5: 恢复流量..."
kubectl patch virtualservice pms-api -n $NAMESPACE \
    --type merge -p '{"spec":{"http":[{"route":[{"destination":{"host":"pms-api","subset":"stable"},"weight":100}]}]}}'

echo "=== 回滚完成 ==="
```
