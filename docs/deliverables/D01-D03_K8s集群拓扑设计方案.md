## 1. 设计概述

### 1.1 设计目标

构建企业级Kubernetes集群，支持AI选品系统的生产运行，满足以下要求：

- **高可用性**: SLA ≥ 99.9%
- **可扩展性**: 支持水平扩展至100+节点
- **GPU支持**: 支持A100/A10 GPU调度
- **多租户**: 支持200+租户隔离
- **安全合规**: 满足企业安全基线

### 1.2 技术选型

| 组件 | 选型 | 版本 | 说明 |
|------|------|------|------|
| 容器运行时 | containerd | 1.7+ | 高性能容器运行时 |
| CNI | Calico | 3.26+ | 支持NetworkPolicy |
| CSI | 本地SSD + S3 | - | 混合存储方案 |
| GPU插件 | NVIDIA Device Plugin | 0.14+ | GPU调度支持 |
| Ingress | Kong Gateway | 3.4+ | API网关 |

### 3.2 网络配置

| 配置项 | 值 | 说明 |
|--------|-----|------|
| Pod CIDR | 10.244.0.0/16 | Pod IP地址池 |
| Service CIDR | 10.96.0.0/12 | Service IP地址池 |
| CNI插件 | Calico | 支持NetworkPolicy |
| IP模式 | BGP | 跨节点Pod通信 |

### 3.3 NetworkPolicy设计

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: pms-api-policy
  namespace: pms-api
spec:
  podSelector: {}
  policyTypes:
    - Ingress
    - Egress
  ingress:
    - from:
        - namespaceSelector:
            matchLabels:
              name: pms-api
        - namespaceSelector:
            matchLabels:
              name: pms-agent
      ports:
        - protocol: TCP
          port: 8000
  egress:
    - to:
        - namespaceSelector:
            matchLabels:
              name: fms-data
      ports:
        - protocol: TCP
          port: 5432
```

## 6. GPU调度方案

### 6.1 GPU资源管理

```yaml
apiVersion: v1
kind: Node
metadata:
  name: gpu-node-1
  labels:
    nvidia.com/gpu.present: "true"
    nvidia.com/gpu.product: "A100-SXM4-80GB"
    nvidia.com/gpu.count: "2"
    nvidia.com/gpu.memory: "81920"
```

### 6.2 MIG配置

```yaml
apiVersion: config.k8s.io/v1
kind: MigConfig
metadata:
  name: mig-config
spec:
  migStrategy: mixed
  devices:
    - deviceType: A100-SXM4-80GB
      migEnabled: true
      migDevices:
        1g.10gb: 7
        2g.20gb: 3
        3g.40gb: 2
        7g.80gb: 1
```

### 6.3 GPU调度策略

```yaml
apiVersion: scheduling.k8s.io/v1
kind: PriorityClass
metadata:
  name: high-priority-gpu
value: 1000000
globalDefault: false
description: "高优先级GPU任务"
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: llm-service
spec:
  template:
    spec:
      containers:
        - name: vllm
          resources:
            limits:
              nvidia.com/gpu: 1
            requests:
              nvidia.com/gpu: 1
      priorityClassName: high-priority-gpu
```
