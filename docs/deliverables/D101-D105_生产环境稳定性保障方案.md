### 2.1 服务等级目标(SLO)

```yaml
服务等级目标:
  可用性:
    目标: 99.9%
    计算方式: (总时间 - 故障时间) / 总时间
    
  响应时间:
    P50: <50ms
    P95: <100ms
    P99: <200ms
    
  错误率:
    目标: <0.1%

  吞吐量:
    目标: ≥1000 QPS
    峰值: ≥2000 QPS
```

### 2.2 错误预算

```python
class ErrorBudget:
    def __init__(self, slo: float = 0.999, period_days: int = 30):
        self.slo = slo
        self.period_days = period_days
        self.total_minutes = period_days * 24 * 60
    
    @property
    def allowed_downtime(self) -> float:
        return self.total_minutes * (1 - self.slo)
    
    @property
    def current_budget(self) -> float:
        used = self.get_downtime_this_period()
        return self.allowed_downtime - used
    
    def get_downtime_this_period(self) -> float:
        return 0
    
    def budget_status(self) -> dict:
        budget = self.current_budget
        total = self.allowed_downtime
        
        return {
            "total_budget_minutes": total,
            "remaining_budget_minutes": budget,
            "budget_consumed_percent": (total - budget) / total * 100,
            "status": "healthy" if budget > total * 0.3 else "warning"
        }
```

class AlertNotification:
    def __init__(self):
        self.channels = {
            "email": EmailChannel(),
            "slack": SlackChannel(),
            "sms": SMSChannel(),
            "phone": PhoneChannel()
        }
    
    async def send(self, alert: Alert):
        severity = alert.severity

            await self.channels["phone"].send(alert)
            await self.channels["slack"].send(alert)
            await self.channels["email"].send(alert)
        elif severity == "warning":
            await self.channels["slack"].send(alert)
            await self.channels["email"].send(alert)
        else:
            await self.channels["email"].send(alert)

### 5.3 故障处理脚本

```bash
#!/bin/bash
# 故障处理脚本

INCIDENT_ID=$1
ACTION=$2

case $ACTION in
    "diagnose")
        echo "=== 诊断故障 ==="
        kubectl get pods -n fms-production
        kubectl logs -l app=pms-api --tail=100 -n fms-production
        ;;
    
    "restart")
        echo "=== 重启服务 ==="
        kubectl rollout restart deployment/pms-api -n fms-production
        ;;
    
    "scale")
        echo "=== 扩容服务 ==="
        kubectl scale deployment/pms-api --replicas=6 -n fms-production
        ;;
    
    "rollback")
        echo "=== 回滚服务 ==="
        kubectl rollout undo deployment/pms-api -n fms-production
        ;;
    
    *)
        echo "Usage: $0 <incident_id> <diagnose|restart|scale|rollback>"
        ;;
esac
```

## 6. 容量管理

### 6.1 容量规划

```python
class CapacityPlanner:
    def __init__(self):
        self.growth_rate = 0.1
        self.safety_margin = 1.3
    
    def predict_capacity(self, current_usage: dict, months: int = 6) -> dict:
        predictions = {}
        
        for resource, usage in current_usage.items():
            predicted = usage * (1 + self.growth_rate) ** months
            recommended = predicted * self.safety_margin
            predictions[resource] = {
                "current": usage,
                "predicted": predicted,
                "recommended": recommended
            }
        
        return predictions
    
    def check_capacity_alert(self, current_usage: dict, limits: dict) -> list:
        alerts = []
        
        for resource, usage in current_usage.items():
            limit = limits.get(resource, 100)
            utilization = usage / limit
            
            if utilization > 0.8:
                alerts.append({
                    "resource": resource,
                    "utilization": utilization,
                    "severity": "warning" if utilization < 0.9 else "critical"
                })
        
        return alerts
```

## 7. 变更管理

### 7.1 变更类型

| 类型 | 定义 | 审批流程 | 风险等级 |
|------|------|---------|---------|
| 标准变更 | 常规操作，低风险 | 自动审批 | 低 |
| 正常变更 | 常规发布，中风险 | 技术负责人审批 | 中 |
| 紧急变更 | 紧急修复，高风险 | 技术负责人+产品负责人审批 | 高 |
