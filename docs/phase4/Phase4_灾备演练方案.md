## 4. 演练执行

### 4.1 数据库故障演练

```bash
# 步骤1: 检查当前主节点
kubectl exec -it postgres-0 -n fms-production -- psql -c "SELECT pg_is_in_recovery();"

# 步骤2: 模拟主节点故障
kubectl delete pod postgres-0 -n fms-production --force --grace-period=0

# 步骤3: 观察故障转移
kubectl get pods -n fms-production -w

# 步骤4: 验证新主节点
kubectl exec -it postgres-1 -n fms-production -- psql -c "SELECT pg_is_in_recovery();"

# 步骤5: 验证应用连接
curl https://api.fms.example.com/health
```

### 4.2 Redis故障演练

```bash
# 步骤1: 检查集群状态
redis-cli -h redis-0 cluster info

# 步骤2: 模拟主节点故障
redis-cli -h redis-0 DEBUG SEGFAULT

# 步骤3: 观察故障转移
redis-cli -h redis-1 cluster info

# 步骤4: 验证服务正常
curl https://api.fms.example.com/health
```

### 4.3 服务故障演练

```bash
# 步骤1: 检查当前Pod分布
kubectl get pods -n fms-production -o wide

# 步骤2: 模拟节点故障
kubectl cordon worker-node-1
kubectl drain worker-node-1 --ignore-daemonsets --delete-emptydir-data

# 步骤3: 观察Pod迁移
kubectl get pods -n fms-production -w

# 步骤4: 验证服务正常
curl https://fms.example.com/health

# 步骤5: 恢复节点
kubectl uncordon worker-node-1
```
