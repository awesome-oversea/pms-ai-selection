# 第一梯队验收标准

## 1. 适用范围

本文对应 `docs/第一梯队可执行开发任务清单.md`，覆盖以下任务的验收口径：

1. 建立正式前端工程与 BFF 层
2. 引入真实 API 网关治理
3. 建立 Elasticsearch/OpenSearch 关键词检索层
4. 落地一个真实 ERP/业务系统集成样板
5. 蓝图能力与当前能力双轨文档机制

---

## 2. T1 建立正式前端工程与 BFF 层

### 验收目标
形成独立前端工程与 BFF 层，至少 1 个工作台真实运行。

### 交付物
- 独立前端工程目录
- BFF 路由层
- 至少 1 个正式工作台页面
- 前后端接口契约说明

### 验收项
- 前端不再依赖后端模板拼装核心业务逻辑
- BFF 具备鉴权透传、错误映射、接口代理能力
- 至少 1 个工作台完成任务创建/查询真实闭环
- 401/403/429/503 错误在页面与 BFF 层表现稳定

### 测试与证据
- 页面联调截图或录屏
- 任务创建与查询联调用例
- BFF 接口测试结果

### 通过标准
- 前端工程、BFF、正式工作台三者同时存在
- 至少 1 条真实业务链路可演示

---

## 3. T2 引入真实 API 网关治理

### 验收目标
形成真实网关层，明确 internal / bff / openapi 边界。

### 交付物
- 网关配置
- API 分层说明
- 限流/版本/消费方治理规则

### 验收项
- internal / bff / openapi 三层边界清晰
- 至少 1 组真实路由经过网关治理生效
- 开放接口具备访问控制、限流、版本规则

### 测试与证据
- 网关配置文件/控制台截图
- 429/401/403 行为验证结果
- OpenAPI 边界元数据

### 通过标准
- 真实网关规则对至少 1 组接口生效
- 开放接口治理规则可重复验证

---

## 4. T3 建立 Elasticsearch/OpenSearch 关键词检索层

### 验收目标
形成正式关键词召回层，与向量召回构成混合检索。

### 交付物
- ES/OpenSearch 索引 mapping
- 写入/更新/删除同步逻辑
- 检索融合逻辑
- 效果验证报告

### 验收项
- 向量召回与 ES/OpenSearch 关键词召回同时可用
- tenant / kb / doc 过滤有效
- 文档更新后关键词索引同步更新
- 至少 1 组查询效果优于当前基线

### 测试与证据
- 检索对比样例
- 同步链路测试结果
- 指标或日志证据

### 通过标准
- 混合检索真实可运行
- 过滤与同步链路稳定

---

## 5. T4 落地一个真实 ERP/业务系统集成样板

### 验收目标
建立至少 1 个真实 ERP/业务系统双向集成样板。

### 交付物
- 集成配置
- 字段映射与主权说明
- 入站/出站同步链路
- 审计与失败处理说明

### 验收项
- 至少 1 条入站 + 1 条出站链路真实可运行
- 幂等、重试、审计、状态查询完整
- 同步失败可人工处理

### 测试与证据
- 样板同步记录
- 审计日志
- 失败重试与人工处理示例

### 通过标准
- 至少 1 个真实系统形成样板闭环

---

## 6. 横向保障项：双轨文档机制

### 验收项
- 当前能力边界文档持续维护
- 目标态蓝图文档持续维护
- 进阶任务清单持续维护
- 重大能力变更可同步反映到三类文档

### 通过标准
- 三类文档均存在且版本一致

---

## 7. 第一梯队总体通过标准

满足以下条件即视为通过：
- T1~T4 各自交付物齐备
- 至少 1 个正式工作台、1 套真实网关治理、1 套正式关键词检索、1 个真实集成样板已跑通
- 双轨文档机制长期可维护
- 发布前具备可演示、可验证、可审计的证据链
