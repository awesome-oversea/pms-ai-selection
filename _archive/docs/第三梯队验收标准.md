# 第三梯队验收标准

## 1. 适用范围

本文对应 `docs/第三梯队可执行开发任务清单.md`，覆盖以下任务的验收口径：

1. 构建数据湖/批流一体样板
2. 建立容量压测与 SLO 统计闭环
3. 提升 Agent 平台能力

---

## 2. T7 构建数据湖/批流一体样板

### 验收目标
建立最小数据资产平台样板，形成离线快照与实时流处理基础。

### 交付物
- 样板数据域定义
- 离线快照链路
- 实时流式链路
- 字段口径/分区/消费说明

### 验收项
- 至少 1 类业务数据形成可追溯数据资产样板
- 至少 1 条实时样板链路可运行
- 流批链路主键与时间语义一致
- 下游至少 1 条消费查询正确返回结果

### 测试与证据
- 快照落库/落湖结果
- 实时链路运行结果
- 字段字典与消费示例

### 通过标准
- 离线 + 实时样板均可运行
- 口径说明可被分析/BI 团队消费

---

## 3. T8 建立容量压测与 SLO 统计闭环

### 验收目标
形成从压测、容量结论到 SLO 指标与告警联动的闭环。

### 交付物
- 压测脚本
- 压测报告模板
- 容量基线结论
- SLI/SLO 定义
- 告警联动配置

### 验收项
- 至少 1 套 API 压测场景可重复执行
- 至少 1 套异步/Worker 链路压测可运行
- 能输出 p95/p99、错误率、吞吐量
- 有扩容建议和告警阈值
- SLO 结果可接入看板

### 测试与证据
- 压测脚本与执行结果
- 容量分析结论
- SLO 看板或告警规则截图/配置

### 通过标准
- 压测、容量、SLO、告警四者形成闭环

---

## 4. T9 提升 Agent 平台能力

### 验收目标
把 Agent 从“任务内编排能力”提升为“平台能力”。

### 交付物
- Agent 运行拓扑模型
- 恢复/重试/人工介入能力
- 运行操作台或等价接口
- 策略版本化与回滚能力

### 验收项
- 至少 1 个多 Agent 任务支持失败后恢复
- 至少 1 个 Agent 策略版本可切换与回滚
- 存在运行拓扑可观察视图或接口
- 失败任务具备人工介入与审计记录
- 运行分析可追踪到节点、阶段、错误与回滚

### 测试与证据
- 恢复与重试演示
- 策略切换记录
- 操作台截图或接口结果
- 审计与指标证据

### 通过标准
- 至少实现“恢复 / 诊断 / 策略切换”中的两项真实闭环
- Agent 平台具备最小运维操作能力

---

## 5. 第三梯队总体通过标准

满足以下条件即视为通过：
- 数据资产样板可运行
- 压测与 SLO 闭环可执行
- Agent 平台能力从“功能”提升到“可观测、可恢复、可治理”的平台态
- 三项能力均具备可审计、可验证、可重复执行的证据链
