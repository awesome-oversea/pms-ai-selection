# 第二梯队验收标准

## 1. 适用范围

本文对应 `docs/第二梯队可执行开发任务清单.md`，覆盖以下任务的验收口径：

1. 拆分 RAG / LLM 为独立服务
2. 建立 Kafka/CDC 驱动的数据同步样板

---

## 2. T5 拆分 RAG / LLM 为独立服务

### 验收目标
在不破坏当前单体能力的前提下，建立 `rag-service` 与 `llm-service` 的独立服务化样板。

### 交付物
- RAG / LLM 服务边界定义
- 服务调用契约
- 单体兼容适配层
- 灰度切换与回退配置
- 部署与观测方案

### 验收项
- 至少 1 项能力完成独立部署样板
- 单体支持 `in-process` / `remote-service` 双模式切换
- request_id / trace_id 可透传
- 超时、重试、熔断、回退可验证
- 预算、配额、审计、指标在拆分后不丢失

### 测试与证据
- 服务联调结果
- 灰度切换验证记录
- 超时回退与错误处理记录
- 监控/审计证据

### 通过标准
- 至少 1 项服务可独立部署且可通过兼容层切换
- 有明确回滚策略与灰度证据

---

## 3. T6 建立 Kafka/CDC 驱动的数据同步样板

### 验收目标
为 1~2 类关键实体建立正式事件驱动同步样板，具备幂等、重试、DLQ、重放能力。

### 交付物
- Topic / 事件 schema / 版本定义
- 生产者/消费者样板
- DLQ / 重试 / 重放机制
- 审计与状态查询接口

### 验收项
- 至少 1 条真实样板链路可运行
- 同一事件重复投递不造成重复写入
- 失败后进入重试或 DLQ，可人工重放
- 租户隔离在事件链路中可验证
- 事件版本具备兼容策略

### 测试与证据
- 端到端同步记录
- DLQ / 重放截图或日志
- 事件 schema 文档
- 审计日志与指标数据

### 通过标准
- 至少 1 条事件同步链路形成完整闭环
- 幂等、重试、DLQ、重放、审计全部有证据

---

## 4. 第二梯队总体通过标准

满足以下条件即视为通过：
- RAG/LLM 至少一项服务独立化成功
- Kafka/CDC 至少一条样板链路稳定可运行
- 单体兼容层、灰度切换、回滚策略完备
- 服务拆分和事件链路均具备观测、审计和错误处理闭环
