## 2. 排序原则

### 2.1 优先级规则

- **P0**：直接影响选品主业务闭环，且现有代码已有基础，建议优先进入最近迭代
- **P1**：直接影响利润飞轮或关键业务扩展，但依赖真实数据源或跨系统联动
- **P2**：产品化与可视化能力，影响可用性和业务呈现
- **P3**：平台化、生产化、长期演进能力

### 2.2 实现难度规则

- **L**：低到中，当前代码已有明显基础，改造集中在单条业务链路或少量模块
- **M**：中等，涉及多模块联动或需要补充真实数据/真实接口
- **H**：高，涉及跨系统闭环、基础设施或生产环境能力

### 2.3 排序方法

先按业务优先级排序，再在同优先级内按难度从低到高排序。

## 5. 详细任务清单

## T01 智能选品统一决策输出层

- **优先级**：P0
- **难度**：L
- **任务说明**：
  将现有 `DataCollection / MarketInsight / ProductPlanner / Commercial` 的阶段性输出收敛为统一决策结果，形成可稳定消费的选品结论结构，包括推荐商品、推荐理由、风险提示、供应链建议、利润区间、定价建议、待确认事项。
- **依赖关系**：无
- **验收标准**：
  1. 选品结果 API 返回统一结构化结果，而不是分散阶段输出拼接。
  2. 结果中至少包含：推荐结论、风险项、利润项、定价项、供应链项、证据来源项。
  3. 选品任务详情页/BFF 可直接消费该统一结构。
  4. 至少补充 1 组接口测试和 1 组服务层测试。

---

## T02 利润 / ROI / 定价优化引擎正式化

- **优先级**：P0
- **难度**：M
- **任务说明**：
  将现有 `CommercialAgent`、成本计算、ROI 预测、定价建议能力从样板算法升级为正式业务引擎，支持明确成本项、利润目标、价格策略和多场景输出。
- **依赖关系**：T01
- **验收标准**：
  1. 支持至少 3 类成本项输入（采购、物流、平台/营销或等价分类）。
  2. 支持至少 3 种输出场景：保守、基准、激进。
  3. 结果进入统一选品决策输出层。
  4. 对利润率、ROI、建议售价补充可重复验证的自动化测试。

---

## T03 BI 正式消费闭环

- **优先级**：P0
- **难度**：M
- **任务说明**：
  将选品结果、利润测算结果、核心指标沉淀到 BI 可消费的数据出口，形成任务级、商品级、利润级指标链路，支撑后续大盘和经营分析。
- **依赖关系**：T01、T02
- **验收标准**：
  1. 至少定义 1 套可稳定消费的 BI 数据集或接口。
  2. 每个选品任务可按任务 ID 回溯核心指标。
  3. BI 输出至少覆盖推荐数量、风险等级、利润率、ROI、价格建议等关键字段。
  4. 至少补充 1 组数据一致性校验测试。

---

## T04 客户反馈反哺闭环（第一版）

- **优先级**：P0
- **难度**：M
- **任务说明**：
  将 CRM/售后/评论类反馈归一化后反哺到选品规则、产品规划和定价建议中，先做第一版闭环，不追求一次做成完整飞轮。
- **依赖关系**：T01
- **验收标准**：
  1. 至少接入 1 类客户反馈来源并完成标准化存储。
  2. 反馈结果可生成可解释的反馈评分或标签。
  3. 反馈标签能进入选品决策结果或定价建议逻辑。
  4. 至少有 1 组端到端测试覆盖“反馈进入 -> 结果变化”。

---

## T05 外部真实数据源最小接入

- **优先级**：P0
- **难度**：M
- **任务说明**：
  将当前趋势/竞品相关 mock 或 simulated 数据替换为最小可用的真实接入，优先打通至少 3 个外部数据源，为趋势预测和竞品监控提供正式底座。
- **依赖关系**：无
- **验收标准**：
  1. 至少 3 个真实外部数据源完成接入、配置和调用验证。
  2. 接口失败时具备重试、降级或明确错误返回。
  3. 运行日志中可区分真实调用与 mock 调用。
  4. 至少有 1 份环境验证记录可证明真实数据可取回。

## T06 市场趋势预测正式落地

- **优先级**：P1
- **难度**：M
- **任务说明**：
  基于真实数据源，形成正式趋势识别与预测链路，输出 7/14/30 天趋势判断、推荐指数，并反哺选品流程。
- **依赖关系**：T05
- **验收标准**：
  1. 支持至少 7/14/30 天三档趋势输出。
  2. 支持趋势评分、趋势方向和商品匹配度输出。
  3. 趋势结果能进入选品决策输出层。
  4. 至少补充 1 组回归测试覆盖趋势结果结构和反哺逻辑。

## T08 WMS 正式业务闭环

- **优先级**：P1
- **难度**：H

## T09 选品到多系统利润飞轮打通

- **优先级**：P1
- **难度**：H
- **任务说明**：
  将选品结果真正打通到 SCM / WMS / OMS / CRM / FMS / BI 的核心链路，形成可回流的利润飞轮，而不是多系统样板并列存在。
- **依赖关系**：T02、T03、T04、T08
- **验收标准**：
  1. 至少完成 1 条端到端业务样例：选品 -> 执行 -> 数据回流 -> 指标复算。
  2. 回流结果能影响下一次选品决策或评分。
  3. 全链路具备 trace / 审计 / 同步状态可查能力。
  4. 至少补充 1 组端到端集成测试或可复演脚本。
