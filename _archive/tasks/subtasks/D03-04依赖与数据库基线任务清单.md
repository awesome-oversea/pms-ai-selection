## D03 依赖清单收口

- 盘点源码中的三方 `import`，对照 `pyproject.toml`、`requirements.txt`、`uv.lock` 查缺补漏。
- 补齐缺失依赖，至少覆盖 `fastapi`、`pytest`、`aiokafka`、`qdrant-client`、`numpy`。
- 清理重复声明和版本漂移，明确单一依赖来源。
- 做一次基础安装与 import smoke test，验证新环境最小可运行。

## D04 ORM Base 与 Session 工厂统一

- 将 SQLAlchemy Declarative Base 收敛为单一定义，建议放在 `src/models/base.py`。
- 修改模型和 `init_db()`，确保建表使用真实业务模型的 metadata。
- 移除 `src/infrastructure/database.py` 中重复 `Base`。
- 清理 `AsyncSessionLocal` / `SessionLocal` 的 `property` 式导出，改为明确的 session factory 接口。

## 产出物

- 修正后的依赖文件
- 安装与 import 验证记录
- 统一后的 ORM Base
- 清晰的 Session 工厂导出方式
