# D17-18 服务解耦与测试重建任务清单

> 来源文档: [问题与建议.md](../../问题与建议.md)
> 总清单: [子任务清单.md](../../子任务清单.md)
> 时间范围: D17-D18
> 目标: 降低 Web/API 与业务实现的直接耦合，并重建最小可信测试体系

## D17 服务层补齐与表现层解耦

- 新增 `ApplicationService` 或 `QueryService` 层承接任务查询与统计逻辑。
- 让 Web 路由不再直接依赖 API endpoint 内部变量。
- 收敛 API 层和 Web 层的重复查询逻辑。
- 明确表现层、应用层、仓储层的职责边界。

## D18 测试体系重建

- 修复与当前实现漂移的测试。
- 重写或新增 settings、repository、health、selection 主链路、RAG 查询等关键测试。
- 删除明显无效或误导性的存在性测试。
- 保证测试断言对齐真实契约，而不是历史实现。

## 产出物

- 应用服务层
- 解耦后的 Web/API 查询路径
- 一组最小可信测试集

## 本段验收

- [ ] `src/web/routes.py` 不再直接依赖 `_task_store` 等内部变量。
- [ ] 关键测试覆盖真实业务行为而不是单纯 import。
- [ ] 本地可执行最小可信测试并得到稳定结果。
