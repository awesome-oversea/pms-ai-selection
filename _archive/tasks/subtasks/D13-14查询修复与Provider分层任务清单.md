# D13-14 查询修复与Provider分层任务清单

> 来源文档: [问题与建议.md](../../问题与建议.md)
> 总清单: [子任务清单.md](../../子任务清单.md)
> 时间范围: D13-D14
> 目标: 修复知识库查询链路，并将 mock 与 real 能力彻底分层

## D13 查询链路修复

- 从真实文档源或向量库构造查询上下文，不再临时创建空 retriever。
- 修复 `RetrievalResult` 被当成 `dict` 使用的问题。
- 增加阈值过滤、结果格式化和基础答案拼装逻辑。
- 保证查询结果结构稳定、可序列化、可追踪。

## D14 Provider 分层

- 为数据源、LLM、趋势分析、成本分析等能力定义 provider interface。
- 将随机逻辑迁移到 mock provider。
- 将真实接入逻辑下沉到 real provider。
- 把 `LLMGateway` 的 mock 开关改为显式环境控制。

## 产出物

- 修复后的知识库查询 API
- provider 接口定义
- mock/real 分层后的实现结构

## 本段验收

- [ ] 查询接口可命中文档且不再因类型错误报错。
- [ ] mock 与 real 代码不再混在同一主流程里。
- [ ] provider 切换路径清晰，默认行为不会误伤生产环境。
