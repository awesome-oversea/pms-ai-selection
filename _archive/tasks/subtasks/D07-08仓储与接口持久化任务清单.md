# D07-08 仓储与接口持久化任务清单

> 来源文档: [问题与建议.md](../../问题与建议.md)
> 总清单: [子任务清单.md](../../子任务清单.md)
> 时间范围: D07-D08
> 目标: 让 Selection 主链路脱离进程内内存，建立稳定的 Repository 和 API 数据访问路径

## D07 Repository 收口

- 整理 `SelectionTaskRepository` 接口，统一命名、输入输出和异常约定。
- 补齐创建任务、查询任务、分页列任务、更新状态与阶段、保存结果、保存失败原因等方法。
- 评估是否拆出 `AgentRunRepository` 或审批相关仓储。
- 补充必要的类型标注和返回契约。

## D08 Selection API 去内存化

- 移除 `selection.py` 中 `_task_store` 作为主数据源的职责。
- 将 `create_selection_task()`、`list_selection_tasks()`、`get_task_detail()`、`cancel_task()` 改为基于数据库读写。
- 让后台流程启动前先完成任务记录落库。
- 确保任务查询行为在进程重启后仍可恢复。

## 产出物

- 稳定的任务仓储接口
- 去内存化后的 Selection API
- 基于数据库的任务 CRUD 能力

## 本段验收

- [ ] API 层不再依赖 `_task_store` 之类进程内变量作为主状态源。
- [ ] 任务列表与详情来自数据库且支持重启后查询。
- [ ] 仓储接口足以支撑主链路读写，不需要在 endpoint 中直接拼 SQL。
