# D09-10 工作流契约与状态落库任务清单

> 来源文档: [问题与建议.md](../../问题与建议.md)
> 总清单: [子任务清单.md](../../子任务清单.md)
> 时间范围: D09-D10
> 目标: 统一 SelectionMaster 与 Agent 返回契约，并让工作流执行状态可持续追踪和恢复

## D09 Agent 返回契约统一

- 明确 `BaseAgent.run()` 的标准返回规范。
- 若保留 `AgentResult`，统一在 `SelectionMaster` 中显式读取 `.output`。
- 若改为纯 `dict`，则同步统一所有 Agent 返回格式和类型约束。
- 检查四个阶段 handler 的结果读取与序列化逻辑。

## D10 工作流状态落库

- 在 `_run_selection_workflow()` 关键节点实时更新任务状态和阶段。
- 任务失败时落库错误信息，完成时落库结果摘要与完整结果。
- 验证“创建任务 -> 执行 -> 查询结果”的最小闭环。
- 重点确认重启后任务状态与结果仍可查询。

## 产出物

- 统一的 Agent 返回契约
- 可 JSON 序列化的工作流结果结构
- 工作流状态实时落库机制

## 本段验收

- [ ] `SelectionMaster` 不再把对象误当成 `dict` 使用。
- [ ] 工作流结果可稳定序列化并存入 JSONB。
- [ ] 任务执行过程中数据库中可见状态变化与最终结果。
