# D23-24 Redis幂等与事件总线任务清单

> 来源文档: [问题与建议.md](../../问题与建议.md)
> 总清单: [子任务清单.md](../../子任务清单.md)
> 时间范围: D23-D24
> 目标: 为异步执行补齐幂等、锁和事件驱动能力

## D23 Redis 锁与幂等设计

- 设计任务级锁、防重复提交和重试去重策略。
- 区分“幂等键”“执行锁”“缓存态”的不同用途。
- 将状态竞争和重复消费场景纳入测试。
- 输出失败恢复与超时释放策略。

## D24 Kafka 事件总线接入

- 定义任务创建、开始执行、阶段完成、失败、取消等事件模型。
- 接入 Kafka 生产与消费骨架。
- 让关键任务状态变化可通过事件广播。
- 预留与审计、通知、监控的联动能力。

## 产出物

- Redis 锁与幂等方案
- 任务事件模型
- Kafka 接入骨架

## 本段验收

- [ ] 相同任务不会因重复提交或重复消费被错误执行多次。
- [ ] 关键状态变化可通过事件流对外发布。
- [ ] Redis 与 Kafka 的职责边界清晰。
