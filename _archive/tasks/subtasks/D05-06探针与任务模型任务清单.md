# D05-06 探针与任务模型任务清单

> 来源文档: [问题与建议.md](../../问题与建议.md)
> 总清单: [子任务清单.md](../../子任务清单.md)
> 时间范围: D05-D06
> 目标: 修复生命周期与健康检查链路，并为任务持久化建立统一状态模型

## D05 生命周期与探针修复

- 梳理 `src/main.py` 启动阶段数据库、Redis、Qdrant 的初始化顺序。
- 明确关闭阶段的资源释放逻辑。
- 统一 `/health`、`/ready`、`/live` 的暴露路径，或同步修正 K8s 探针配置。
- 让 readiness 返回真实依赖状态，而不是静态成功。

## D06 任务状态模型设计

- 设计任务主表状态字段，包括 `status`、`phase`、`result_summary`、`execution_result`、`error`、`started_at`、`completed_at`。
- 明确 Agent 执行记录、审批记录是否需要落库。
- 明确 DB 与 Redis 的职责边界，避免状态来源不清。
- 产出任务状态流转图和字段映射说明。

## 产出物

- 生命周期与探针修复方案
- 修正后的 K8s 探针口径
- 任务状态流转图
- 任务表字段设计说明

## 本段验收

- [ ] `/health` 与 `/ready` 路由和部署探针一致。
- [ ] 启动与关闭顺序清晰且可追踪。
- [ ] 任务状态模型能够覆盖执行中、成功、失败、取消等核心场景。
