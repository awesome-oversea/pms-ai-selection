# D07-08 仓储与接口持久化验收标准

> 来源任务清单: [D07-08仓储与接口持久化任务清单](../../subtasks/D07-08仓储与接口持久化任务清单.md)
> 所属总表: [子任务清单.md](../../../子任务清单.md)
> 验收范围: D07-D08
> 验收目标: 建立稳定仓储接口，并让 Selection API 脱离进程内内存状态

## 一、验收范围

本验收标准适用于任务仓储收口和 Selection API 持久化改造阶段。

## 二、核心验收项

| 序号 | 验收项 | 验收标准 | 验收方法 | 结果 |
|------|------|------|------|------|
| A01 | 仓储接口收口 | `SelectionTaskRepository` 覆盖创建、查询、分页、状态更新、结果保存、错误保存等核心方法 | 代码审查 | ☐ |
| A02 | 仓储契约清晰 | 方法命名、输入输出、异常约定和类型标注一致 | 代码审查 | ☐ |
| A03 | API 去内存化 | `selection.py` 不再以 `_task_store` 作为主数据源 | 代码审查 | ☐ |
| A04 | 创建任务落库 | 创建任务时先落库，再启动后台流程 | 接口验证 + 数据验证 | ☐ |
| A05 | 列表与详情查询 | 任务列表和详情改为从数据库读取，支持分页 | 接口验证 | ☐ |
| A06 | 取消任务生效 | `cancel_task()` 会真实更新数据库状态 | 接口验证 + 数据验证 | ☐ |
| A07 | 重启后可查询 | 应用重启后，历史任务仍可通过 API 查询 | 联调验证 | ☐ |
| A08 | 状态一致性 | API 返回结果与数据库状态一致，不出现明显漂移 | 数据核对 | ☐ |

## 三、交付物验收

| 序号 | 交付物 | 必要内容 | 结果 |
|------|------|------|------|
| D01 | 仓储接口说明 | 方法列表、职责边界、异常定义 | ☐ |
| D02 | 持久化后的 Selection API | 创建、列表、详情、取消等接口完成改造 | ☐ |
| D03 | 数据校验记录 | API 结果与数据库记录对照结果 | ☐ |

## 四、通过标准

- 核心验收项 `8/8` 通过。
- 交付物验收 `3/3` 齐全。
- 任务主状态已从进程内内存迁移到数据库。
