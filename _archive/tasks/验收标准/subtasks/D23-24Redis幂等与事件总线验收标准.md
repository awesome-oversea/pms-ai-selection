# D23-24 Redis幂等与事件总线验收标准

> 来源任务清单: [D23-24Redis幂等与事件总线任务清单](../../subtasks/D23-24Redis幂等与事件总线任务清单.md)
> 所属总表: [子任务清单.md](../../../子任务清单.md)
> 验收范围: D23-D24
> 验收目标: 为异步执行补齐 Redis 幂等控制和 Kafka 事件总线能力

## 一、验收范围

本验收标准适用于幂等、锁机制和事件总线接入阶段。

## 二、核心验收项

| 序号 | 验收项 | 验收标准 | 验收方法 | 结果 |
|------|------|------|------|------|
| A01 | 幂等策略设计 | 已明确幂等键、执行锁、缓存态三类用途与边界 | 方案评审 | ☐ |
| A02 | 锁机制有效 | 相同任务不会因重复提交或重复消费被错误执行多次 | 联调验证 | ☐ |
| A03 | 超时释放与重试 | 锁超时释放、失败重试和去重策略明确 | 方案评审 + 运行验证 | ☐ |
| A04 | 事件模型定义 | 任务创建、开始、完成、失败、取消等事件结构清晰 | 方案评审 | ☐ |
| A05 | Kafka 骨架可用 | 已具备最小生产与消费能力，可处理关键任务事件 | 联调验证 | ☐ |
| A06 | 状态事件可发布 | 关键状态变化可以通过事件流对外广播 | 联调验证 | ☐ |
| A07 | 组件边界明确 | Redis 与 Kafka 的职责边界清晰，无重复承担同一职责 | 设计评审 | ☐ |

## 三、交付物验收

| 序号 | 交付物 | 必要内容 | 结果 |
|------|------|------|------|
| D01 | Redis 幂等方案 | 幂等键、锁策略、超时释放、异常处理 | ☐ |
| D02 | 任务事件模型 | 事件类型、字段、发布时机、消费约定 | ☐ |
| D03 | Kafka 接入骨架 | 生产/消费入口、配置说明、验证记录 | ☐ |

## 四、通过标准

- 核心验收项 `7/7` 通过。
- 交付物验收 `3/3` 齐全。
- 异步链路具备基本幂等和事件化能力。
