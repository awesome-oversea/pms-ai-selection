# D09-10 工作流契约与状态落库验收标准

> 来源任务清单: [D09-10工作流契约与状态落库任务清单](../../subtasks/D09-10工作流契约与状态落库任务清单.md)
> 所属总表: [子任务清单.md](../../../子任务清单.md)
> 验收范围: D09-D10
> 验收目标: 统一 Agent 返回契约，并实现工作流状态与结果的全过程落库

## 一、验收范围

本验收标准适用于 BaseAgent、SelectionMaster 和任务工作流状态持久化整改阶段。

## 二、核心验收项

| 序号 | 验收项 | 验收标准 | 验收方法 | 结果 |
|------|------|------|------|------|
| A01 | Agent 返回规范 | 已明确 `BaseAgent.run()` 的统一对外返回规范 | 方案评审 | ☐ |
| A02 | `SelectionMaster` 读取正确 | 不再把对象误当成 `dict`，能正确读取 Agent 输出 | 代码审查 | ☐ |
| A03 | 结果可序列化 | 工作流输出结果可稳定进行 JSON 序列化 | 代码审查 + 单测验证 | ☐ |
| A04 | 结果可落库 | 任务执行结果可安全存储到 JSONB 或等价结构 | 数据验证 | ☐ |
| A05 | 关键状态实时更新 | `_run_selection_workflow()` 在阶段切换时实时更新数据库状态 | 代码审查 + 运行验证 | ☐ |
| A06 | 失败信息落库 | 任务失败时错误信息、失败阶段和时间可追踪 | 数据验证 | ☐ |
| A07 | 完成结果落库 | 任务完成后结果摘要和完整结果均已保存 | 数据验证 | ☐ |
| A08 | 最小闭环验证 | 至少完成一次“创建任务 -> 执行 -> 查询结果”的闭环验证 | 联调验证 | ☐ |

## 三、交付物验收

| 序号 | 交付物 | 必要内容 | 结果 |
|------|------|------|------|
| D01 | Agent 契约说明 | 返回结构、字段语义、序列化规范 | ☐ |
| D02 | 工作流状态落库说明 | 状态更新点、失败记录、完成记录 | ☐ |
| D03 | 闭环验证记录 | 创建、执行、查询的实际验证证据 | ☐ |

## 四、通过标准

- 核心验收项 `8/8` 通过。
- 交付物验收 `3/3` 齐全。
- 工作流结果和状态已具备稳定持久化能力。
