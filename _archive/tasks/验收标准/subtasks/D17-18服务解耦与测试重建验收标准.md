# D17-18 服务解耦与测试重建验收标准

> 来源任务清单: [D17-18服务解耦与测试重建任务清单](../../subtasks/D17-18服务解耦与测试重建任务清单.md)
> 所属总表: [子任务清单.md](../../../子任务清单.md)
> 验收范围: D17-D18
> 验收目标: 降低 Web/API 与内部实现的耦合，并重建最小可信测试体系

## 一、验收范围

本验收标准适用于应用服务层补齐、表现层解耦和测试体系重建阶段。

## 二、核心验收项

| 序号 | 验收项 | 验收标准 | 验收方法 | 结果 |
|------|------|------|------|------|
| A01 | 服务层建立 | 已新增 `ApplicationService` 或 `QueryService` 等服务层承接业务查询与统计逻辑 | 代码审查 | ☐ |
| A02 | Web 层解耦 | `src/web/routes.py` 不再直接依赖 `_task_store`、`_agent_registry` 等内部变量 | 代码审查 | ☐ |
| A03 | API/Web 访问口径统一 | API 层和 Web 层通过统一服务层访问业务数据 | 代码审查 + 联调验证 | ☐ |
| A04 | 查询逻辑收敛 | 同类任务查询或统计逻辑不再在多层重复实现 | 代码审查 | ☐ |
| A05 | 漂移测试修复 | 与当前实现不一致的旧测试已修复或替换 | 测试审查 | ☐ |
| A06 | 关键测试补齐 | settings、repository、health、selection 主链路、RAG 查询等关键测试已建立 | 测试验证 | ☐ |
| A07 | 无效测试清理 | 明显误导或仅验证“能 import”的测试已清理 | 测试审查 | ☐ |
| A08 | 最小可信测试可运行 | 本地可执行一组稳定的最小可信测试集 | 命令验证 | ☐ |

## 三、交付物验收

| 序号 | 交付物 | 必要内容 | 结果 |
|------|------|------|------|
| D01 | 服务层设计说明 | 服务职责、调用边界、使用方式 | ☐ |
| D02 | 解耦改造结果 | Web/API 调用路径说明、替换点清单 | ☐ |
| D03 | 最小可信测试集 | 关键测试文件、运行方式、预期结果 | ☐ |

## 四、通过标准

- 核心验收项 `8/8` 通过。
- 交付物验收 `3/3` 齐全。
- 代码结构与测试体系能够支撑 D19-D20 的 CI 建设。
