# D03-04 依赖与数据库基线验收标准

> 来源任务清单: [D03-04依赖与数据库基线任务清单](../../subtasks/D03-04依赖与数据库基线任务清单.md)
> 所属总表: [子任务清单.md](../../../子任务清单.md)
> 验收范围: D03-D04
> 验收目标: 完成依赖收口、安装验证、ORM Base 统一和 Session 工厂整理

## 一、验收范围

本验收标准适用于依赖声明与数据库基线整改阶段，覆盖依赖清单、安装验证、ORM 元数据统一和数据库会话工厂导出方式。

## 二、核心验收项

| 序号 | 验收项 | 验收标准 | 验收方法 | 结果 |
|------|------|------|------|------|
| A01 | 三方依赖盘点 | 已完成源码三方 `import` 与依赖文件的差异比对 | 文档审查 | ☐ |
| A02 | 依赖声明收口 | `pyproject.toml`、`requirements.txt`、锁文件之间无关键依赖遗漏和明显漂移 | 文件审查 | ☐ |
| A03 | 核心依赖补齐 | `fastapi`、`pytest`、`aiokafka`、`qdrant-client`、`numpy` 等关键依赖已声明 | 文件审查 | ☐ |
| A04 | 安装验证 | 新环境可按清单完成基础安装，不出现关键依赖缺失 | 环境验证 | ☐ |
| A05 | Import 烟雾验证 | 关键模块可完成基础 import，不因依赖缺失失败 | 命令验证 | ☐ |
| A06 | ORM Base 统一 | 业务模型与建表逻辑使用同一份 Declarative Base 和 metadata | 代码审查 | ☐ |
| A07 | `init_db()` 元数据正确 | 自动建表针对真实模型，而不是重复或错误的 Base | 代码审查 + 启动验证 | ☐ |
| A08 | Session 工厂导出清晰 | 不再以 `property` 方式导出 `SessionLocal` / `AsyncSessionLocal` | 代码审查 | ☐ |
| A09 | Session 接口可读 | 提供明确的 session factory 或获取接口，调用方式一致 | 代码审查 | ☐ |
