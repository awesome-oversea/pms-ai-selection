# D13-14 查询修复与Provider分层验收标准

> 来源任务清单: [D13-14查询修复与Provider分层任务清单](../../subtasks/D13-14查询修复与Provider分层任务清单.md)
> 所属总表: [子任务清单.md](../../../子任务清单.md)
> 验收范围: D13-D14
> 验收目标: 修复知识库查询链路，并完成 mock 与 real provider 的职责分层

## 一、验收范围

本验收标准适用于 RAG 查询修复和 provider 分层阶段。

## 二、核心验收项

| 序号 | 验收项 | 验收标准 | 验收方法 | 结果 |
|------|------|------|------|------|
| A01 | 查询上下文来源正确 | 查询时从真实文档源或向量库构造检索上下文 | 代码审查 + 联调验证 | ☐ |
| A02 | 类型误用修复 | `RetrievalResult` 不再被当成 `dict` 直接使用 | 代码审查 | ☐ |
| A03 | 查询结果稳定 | 支持阈值过滤、结果格式化，返回结构稳定可序列化 | 接口验证 | ☐ |
| A04 | Provider 接口定义 | 已为数据源、LLM、趋势、成本等能力定义 interface | 方案评审 | ☐ |
| A05 | Mock 逻辑分层 | 随机逻辑已迁移到 mock provider，不再混入主流程 | 代码审查 | ☐ |
| A06 | Real 逻辑分层 | 真实接入逻辑位于 real provider，路径清晰 | 代码审查 | ☐ |
| A07 | 环境开关明确 | `LLMGateway` 等关键能力的 mock/real 切换由显式配置控制 | 配置审查 + 运行验证 | ☐ |

## 三、交付物验收

| 序号 | 交付物 | 必要内容 | 结果 |
|------|------|------|------|
| D01 | 查询修复说明 | 查询链路、类型修复点、返回结构说明 | ☐ |
| D02 | Provider 设计说明 | 接口定义、mock/real 边界、切换策略 | ☐ |
| D03 | 联调验证记录 | 上传后查询命中、切换模式验证结果 | ☐ |

## 四、通过标准

- 核心验收项 `7/7` 通过。
- 交付物验收 `3/3` 齐全。
- 查询链路可用，provider 分层能支撑 D15-D16 的接入改造。
