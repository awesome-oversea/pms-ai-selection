## 一、项目定位与目标

### 1.1 当前阶段定位

当前项目处于 **单体原型收敛阶段**，已具备选品闭环原型和本地可验收基线，但距离企业级目标态仍有明显差距。

### 1.2 核心目标

**短期目标**：打通 selection 执行闭环专项
- adopt（采纳）
- execution-feedback-sync（执行反馈同步）
- close-loop（闭环）
- close-loop-overview（闭环概览）

**中期目标**：完成 MVP 经营闭环
- 真实品类、真实市场、真实租户场景跑通
- 选品建议、审批、采纳、执行、反馈、知识沉淀、报告输出都依赖真实数据源

**长期目标**：企业化治理与规模化
- 统一身份、统一事件总线、统一可观测、统一部署与容量

### 1.3 当前关键问题

| 问题类型 | 具体问题 | 影响 |
|---------|---------|------|
| 业务前置条件 | 任务多为pending状态，无GO/REVIEW决策结果 | adopt被业务规则拒绝 |
| ERP配置 | SCM/WMS/OMS/CRM/FMS配置不完整 | 闭环动作依赖缺失 |
| 环境稳定性 | Docker/WSL/Qdrant/OpenSearch恢复不一致 | 服务漂移导致测试失败 |

- scope、purpose、data_level、source_system
- request_id、trace_id、idempotency_key
- input_snapshot_hash、output_snapshot_hash、evidence_chain_id、created_at

## 四、数据主权与主数据边界（整合自PMS-ERP数据主权说明.md）

### 4.1 数据主权总原则

1. 正式业务主数据归 ERP
2. 正式业务单据归 ERP
3. 正式业务状态归 ERP
4. 成本、利润、库存、订单、财务、KPI口径归 ERP
5. AI建议、评分、证据链、假设、风险、洞察归 PMS
6. PMS可保存ERP数据快照、特征和分析结果，但不得覆盖ERP真相源
7. PMS输出进入ERP后，只能形成建议、草稿、待审批动作、风险预警或洞察卡片
8. 建议执行状态由ERP控制，建议内容和证据链由PMS控制

### 4.2 数据可信等级

| 等级 | 数据来源 | 决策用途 |
|---|---|---|
| A | ERP订单、库存、财务、采购、BI正式指标 | 经营事实与强约束 |
| A/B | 官方平台API、广告平台API | 业务事实与外部平台表现 |
| B | 第三方数据服务 | 市场容量与趋势参考 |
| C | 爬虫、社媒、新闻、论坛 | 趋势信号与风险提示 |
| D | LLM推断 | 解释、总结、假设，不作事实源 |

## 五、建议执行状态机（整合自PMS建议池与ERP审批状态机设计.md）

### 5.1 状态流转

```text
created → scored → submitted → accepted → pending_approval → approved
                                                        ↓
                                               approval_rejected
                                          ↓
                          executing → partially_executed → executed
                               ↓               ↓
                            failed ←────── rolled_back
                                         ↓
                                    measured → reviewed
```

### 5.2 状态定义

| 状态 | 含义 | 主控系统 | 可逆 |
|---|---|---|---|
| created | PMS创建建议草稿 | PMS | 是 |
| scored | 完成评分和证据链 | PMS | 是 |
| submitted | PMS提交ERP | PMS | 否 |
| accepted | ERP接收并创建建议池记录 | ERP | 否 |
| rejected | ERP接收前拒绝 | ERP | 否 |
| pending_approval | 进入ERP审批流 | ERP | 是 |
| approval_rejected | 审批拒绝 | ERP | 否 |
| approved | 审批通过 | ERP | 条件可逆 |
| executing | ERP开始执行 | ERP | 否 |
| partially_executed | 部分执行 | ERP | 是 |
| executed | 执行完成 | ERP | 条件可回滚 |
| failed | 执行失败 | ERP | 可重试 |
| rolled_back | 已回滚 | ERP | 否 |
| measured | 效果已评估 | ERP BI | 否 |
| reviewed | PMS复盘完成 | PMS | 否 |

| IAM | ERP主控权限 | 授权申请、服务账号注册 | 所有PMS调用必须通过IAM校验 |
| PDM | ERP主控产品立项 | 选品建议、合规风险 | 立项必须经ERP PDM审批 |
| SOM | Listing归SOM | Listing草稿、定价建议 | 上架、改价由ERP SOM审批 |
| ADS | ERP主控投放 | 预算、竞价、关键词建议 | 高风险调整必须审批 |
| OMS | ERP主控订单 | 风险预警、异常识别 | PMS不得修改订单终态 |
| SCM | ERP主控采购 | 采购建议、补货建议 | 正式采购单必须审批 |
| WMS | ERP主控库存 | 库存预测、调拨建议 | PMS不得写库存流水 |
| FBA | ERP主控FBA货件 | FBA补货建议、入仓草稿 | 正式货件必须审批 |
| TMS | ERP主控物流 | 物流风险建议、渠道选择建议 | PMS不得切换正式物流渠道 |
| CRM | ERP主控客服售后 | 情绪分析、产品痛点、售后风险洞察 | PMS不得赔付/退款/关闭工单 |
| FMS | ERP主控财务 | 利润风险、定价利润模拟 | PMS不得写财务凭证 |
| BI | ERP主控KPI口径 | AI洞察卡片、复盘报告 | PMS不得替代BI指标口径 |
| SYS | ERP主控系统配置 | 配置变更申请、灰度建议 | 生产配置变更必须审批 |
| DASHBOARD | ERP主控工作台 | 洞察卡片、建议待办、风险提醒 | 看板展示必须执行IAM权限 |
