### 5.2 时序图详解

#### 业务背景

该时序图描述跨境电商运营人员使用AI选品系统的**完整核心流程**：从发起选品请求，到多Agent协同分析，再到生成推荐报告，最后“一键采纳”驱动ERP系统执行采购、仓储、上架，并实现数据回流与模型自进化。

#### 参与者（角色与系统）





| 参与者                    | 类型      | 职责                                                   |
| :------------------------ | :-------- | :----------------------------------------------------- |
| **运营专员**              | 业务角色  | 输入选品需求、查看推荐、采纳决策                       |
| **前端 (Next.js)**        | 前端系统  | 提供交互界面，推送请求和接收结果                       |
| **API网关 (Kong)**        | 基础设施  | 认证、限流、审计、路由                                 |
| **选品API (FastAPI)**     | 后端服务  | 接收REST请求，管理任务生命周期                         |
| **协调Agent (LangGraph)** | AI编排    | 分解任务，调度专业Agent                                |
| **5个专业Agent**          | AI执行    | 分别负责数据采集、市场洞察、产品规划、商业化、风险评估 |
| **AI中台**                | 能力层    | 统一调用llm、rag、embedding服务                        |
| **LLM模型层**             | 推理层    | 开源/商业大模型，按场景路由                            |
| **RAG知识库**             | 存储/检索 | 向量+关键词混合检索，提供背景知识                      |
| **报告生成Agent**         | AI执行    | 汇总结果，生成结构化报告                               |
| **数据存储层**            | 基础设施  | PostgreSQL、Kafka、数据湖等                            |
| **SCM / WMS / OMS**       | 内部ERP   | 供应链、仓储、订单系统                                 |
| **CRM**                   | 内部系统  | 客户评价与客诉数据                                     |

#### 核心业务步骤与技术映射





| 步骤 | 业务动作                   | 技术组件                      | 技术细节                                                     |
| :--- | :------------------------- | :---------------------------- | :----------------------------------------------------------- |
| 1    | 运营填写需求，提交选品任务 | Next.js + Kong + FastAPI      | 前端表单校验，网关认证限流，API生成唯一task_id               |
| 2    | 异步触发协调Agent          | FastAPI + LangGraph           | API将任务放入Celery/Ray队列，Master Agent初始化状态机        |
| 3    | 并行执行5个专业Agent       | Ray Actor + 各Agent框架       | 利用Ray实现并行，减少总耗时；每个Agent调用AI中台             |
| 3.1  | 数据采集                   | AutoGen + 外部API             | 并发调用Amazon/TikTok/1688/媒体API，写入Kafka和数据湖        |
| 3.2  | 市场洞察                   | LangGraph + Google Trends API | 获取搜索热度，调用LLM分析供需比和生命周期                    |
| 3.3  | 产品规划                   | CrewAI + LLaVA-13B            | 多模态分析竞品图片和TikTok视频，提取差异化卖点               |
| 3.4  | 商业化分析                 | LangGraph + DeepSeek-V3       | 结合FMS成本数据和1688报价，计算利润、定价、ROI               |
| 3.5  | 风险评估                   | GraphRAG + bge-reranker       | 混合检索专利库和媒体资讯，精排后输出风险等级                 |
| 4    | 汇总生成报告               | CrewAI + GPT-4/Qwen2.5-72B    | 将所有Agent结果输入报告Agent，生成PDF/Excel并存储            |
| 5    | 推送结果至前端             | WebSocket / SSE               | 通过WebSocket实时推送推荐列表，前端ECharts渲染               |
| 6    | 运营一键采纳               | FastAPI + SCM/WMS/OMS API     | 调用ERP系统接口，创建采购单、预留库存、生成Listing草稿       |
| 7    | 数据回流与模型自进化       | Flink + Kafka + RAG           | 销售后订单/评价通过CDC进入Kafka，Flink计算特征并更新知识库，定期微调LLM |

#### 关键技术决策体现

- **异步与并行**：选品任务采用异步处理，避免阻塞；五个Agent并行执行，整体耗时从串行10分钟降至2分钟。
- **模型路由**：AI中台根据任务复杂度自动选择LLM（如趋势分析用Qwen2.5-72B，多模态用LLaVA-13B，报告生成用GPT-4），平衡效果与成本。
- **RAG增强**：风险评估Agent通过混合检索（Qdrant向量+ES关键词）获取专利和新闻，再用bge-reranker精排，提高风险识别准确率。
- **ERP闭环**：“一键采纳”直接触发SCM/WMS/OMS API，实现从决策到执行的自动化，缩短采购上架周期。
- **自进化机制**：销售数据通过Kafka回流，Flink实时计算特征，更新RAG知识库和模型训练集，使爆款命中率持续提升。

#### 性能与可靠性说明

- **网关层**：Kong限流保护后端，防止爬虫或异常流量。
- **Agent层**：LangGraph状态机支持断点重试，Human-in-the-loop可在关键节点人工干预。
- **推理层**：vLLM集群支持张量并行，单卡故障自动切换；商业API配置熔断降级。
- **数据层**：PostgreSQL主从复制，Qdrant集群三副本，Kafka持久化保证消息不丢失。
