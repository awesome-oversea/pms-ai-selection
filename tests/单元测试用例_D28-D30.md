## 核心组件说明

### Web路由 (`src/web/routes.py`)
- **GET /dashboard**: 仪表盘首页
- **GET /selection**: 选品任务面板
- **GET /approval**: 审批管理界面
- **GET /results/{task_id}**: 选品结果展示
- **GET /agents/monitor**: Agent监控面板

### HTML模板 (`web/templates/`)
- `dashboard.html`: 系统仪表盘 - 统计卡片、最近任务、快速操作
- `selection.html`: 选品面板 - 创建表单、任务列表、状态筛选
- `approval.html`: 审批管理 - 待审批、审批历史、规则配置
- `result.html`: 结果展示 - 决策Banner、市场洞察、产品规划、商业化评估
- `agents.html`: Agent监控 - 状态总览、健康检查、详情查看

### 样式资源 (`web/static/css/main.css`)
- 响应式布局系统(CSS Grid/Flexbox)
- 组件样式: navbar, card, btn, badge, stat-card, data-table, form-group
- 决策Banner: go(绿)/conditional_go(黄)/no_go(红)
- 移动端适配(@media max-width:768px)

### JavaScript模块 (`web/static/js/`)
- `main.js`: 公共工具函数(apiRequest, showToast, formatTime)
- `selection.js`: 选品面板逻辑(创建任务、取消任务、状态刷新)
- `approval.js`: 审批界面逻辑(审批统计、审批操作)
- `result.js`: 结果页动画(进度条模拟)
- `agents.js`: Agent监控逻辑(健康检查、详情展示、测试调用)

## 技术实现要点

### Jinja2模板引擎配置
```python
_jinja_env = Environment(
    loader=FileSystemLoader(templates_dir),
    autoescape=select_autoescape(['html', 'xml']),
)
```
- 使用自定义Environment替代FastAPI的Jinja2Templates
- 解决了Starlette/Jinja2Templates缓存导致的"unhashable type: 'dict'"问题
- 启用自动转义防止XSS攻击

### 静态文件挂载
```python
app.mount("/static", StaticFiles(directory=static_dir), name="static")
```
- 通过main.py统一挂载静态资源目录
- 支持CSS/JS/图片等静态资源访问

### 根路径重定向
```python
@router.get("/")
async def root():
    return RedirectResponse(url="/dashboard")
```
- 访问根路径自动跳转到仪表盘

## 已修复问题清单

| 问题 | 原因 | 修复方案 |
|------|------|---------|
| unhashable type: 'dict' | Jinja2Templates缓存机制与dict冲突 | 使用自定义Environment + cache_size=0 |
| takes 1 positional argument | 模块缓存导致旧签名残留 | 重写文件，使用_render辅助函数 |
| 路径计算错误 | dirname层级不匹配(应为2层而非3层) | 统一使用os.path.dirname(os.path.abspath(__file__))两层 |
| selectattr过滤器错误 | dict列表无法使用selectattr | 移除复杂过滤器，简化为占位符 |
| OPTIONS方法405 | FastAPI默认不支持OPTIONS | 改用GET请求测试CORS |
