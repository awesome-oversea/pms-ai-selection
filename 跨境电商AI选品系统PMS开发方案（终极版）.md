## 1. 项目总览：感知、记忆与决策的协同模型

### 1.1 系统定位与核心价值





| 传统 PMS 系统              | **本 AI 选品系统**            |
| :------------------------- | :---------------------------- |
| **记录**商品信息           | **决定**做什么商品            |
| **被动**响应查询           | **主动**发起决策              |
| 独立于业务流的**信息孤岛** | 深度嵌入ERP流程的**操作系统** |
| 依赖个人经验的**艺术**     | 基于数据与AI的**科学**        |

### 1.2 感知-记忆-决策协同模型

本系统的核心运行逻辑可以抽象为**感知-记忆-决策**三位一体的协同模型：





| 角色                  | 对应系统        | 核心职责                                                     | **在选品场景中的类比**                                       |
| :-------------------- | :-------------- | :----------------------------------------------------------- | :----------------------------------------------------------- |
| **感知 (Perception)** | **数据中台**    | 实时捕获、清洗、加工企业内部（ERP）与外部（API/爬虫）的**现状与信号**。 | 选品专家的“眼睛和耳朵”，持续观察市场动态、竞品动作、自身库存。 |
| **记忆 (Memory)**     | **知识库**      | 存储、组织、检索企业内外部的**历史经验与结构化知识**。       | 选品专家的“大脑皮层”，存储过往成功/失败案例、行业规则、产品标准。 |
| **决策 (Decision)**   | **Agent编排层** | 基于**感知**数据和**记忆**知识，通过逻辑推理与模型计算，生成**行动指令**。 | 选品专家的“前额叶皮层”，综合分析信息，做出选品判断并驱动执行。 |

## 3. 【核心】数据中台、知识库与Agent的协同机制

### 3.1 三者角色与关系模型





| 维度              | **数据中台 (感知)**                            | **知识库 (记忆)**                                    | **Agent (决策)**                   |
| :---------------- | :--------------------------------------------- | :--------------------------------------------------- | :--------------------------------- |
| **核心问题**      | **“正在发生什么？”**                           | **“过去发生过什么？规则是什么？”**                   | **“我应该做什么？”**               |
| **数据特征**      | 实时、动态、海量、原始                         | 静态、结构化、高质量、已验证                         | 逻辑性、目标导向                   |
| **典型内容**      | 竞品实时价格、最新评价、BSR排名变化            | 选品成功案例、产品设计规范、专利全文、供应商画像     | 选品推荐列表、定价策略、采购计划   |
| **更新频率**      | 秒级 ~ 小时级                                  | 天级 ~ 周级                                          | 任务触发时                         |
| **对Agent的价值** | 提供**现状**与**信号**，用于识别机会与风险。   | 提供**经验**与**规则**，用于指导决策与规避错误。     | 输出**行动方案**，驱动业务执行。   |
| **来自ERP/外部**  | **OMS**销量、**CRM**新评价、**Amazon**价格变动 | **FMS**利润模型、**SCM**优秀供应商档案、**法规**文件 | **SCM**采购单、**OMS** Listing草稿 |

### 3.2 协同交互协议与接口设计

为确保三者高效协同，定义了标准化的交互接口：

#### 3.2.1 Agent → 数据中台 (查询特征)

- **接口**：`FeatureStore.get_online_features(feature_names: List[str], entity_keys: Map[str, Value])`
- **示例**：市场洞察Agent查询商品 `ASIN=B08XYZ` 的特征。

python

```
# Agent调用
features = feature_store.get_online_features(
    feature_names=["sales_growth_rate_7d", "price_volatility_30d", "review_sentiment_score"],
    entity_keys={"asin": "B08XYZ"}
)
# 返回: {"sales_growth_rate_7d": 0.23, "price_volatility_30d": 0.05, "review_sentiment_score": 0.8}
```



#### 3.2.2 Agent → 知识库 (检索知识)

- **接口**：`RAGService.retrieve(query: str, top_k: int, filters: Dict)`
- **示例**：产品规划Agent检索“户外电源 轻量化设计 方案”。

python

```
# Agent调用
knowledge_chunks = rag_service.retrieve(
    query="户外电源 轻量化设计 成功案例",
    top_k=5,
    filters={"doc_type": "case_study", "category": "outdoor_power"}
)
```



#### 3.2.3 Agent → 知识图谱 (查询关系)

- **接口**：`GraphService.query(cypher_query: str)`
- **示例**：风险评估Agent查询与“21700电芯”相关的专利侵权风险。

cypher

```
// Agent调用
result = graph_service.query("""
    MATCH (c:Component {name: '21700 Battery Cell'})-[r:POTENTIAL_INFRINGEMENT]->(p:Patent)
    RETURN p.patent_id, p.owner, p.title
""")
```



#### 3.2.4 数据中台 → 知识库 (沉淀更新)

- **机制**：**Flink流处理任务**或**Spark批处理任务**消费处理后的数据，提取有价值的信息，写入知识库。
- **示例**：Flink任务发现某个商品的“充电慢”负面评价激增，将该评价片段及情感标签作为一条新的知识条目，通过API写入`rag-service`的待审核队列。

### 3.4 协同价值场景化呈现





| 场景           | **仅用数据中台 (无知识库)**                  | **仅用知识库 (无实时数据)**                      | **三者协同 (本方案)**                                        |
| :------------- | :------------------------------------------- | :----------------------------------------------- | :----------------------------------------------------------- |
| **竞品价格战** | 感知到降价，但不知如何应对。                 | 知道“降价应对策略”文档，但不知该何时触发。       | **实时感知降价信号，自动检索“价格战应对SOP”知识，Agent生成“跟进降价5%或捆绑赠品”的具体建议。** |
| **差评突增**   | 感知到差评，但无法快速定位根本原因。         | 知道某类产品常见差评原因，但不知当前是否发生。   | **实时感知差评模式，检索知识库中“同类产品设计缺陷案例”，Agent判断可能是设计问题并发出预警。** |
| **新品选品**   | 能看到市场数据好，但无法判断长期潜力与风险。 | 知道选品的一般原则，但无法判断当前市场是否饱和。 | **结合实时供需比特征和历史选品成功/失败案例，Agent给出“短期有机会但长期竞争激烈”的综合判断。** |

#### 5.3.2 Elasticsearch 关键词检索详细设计





| 设计维度         | 具体配置                                                     | **业务能力体现**                                   |

#### 5.3.7 RRF融合算法详细实现

python

```
def rrf_fusion(
    es_results: List[Dict],
    qdrant_results: List[Dict],
    k: int = 60,
    es_weight: float = 1.0,
    qdrant_weight: float = 1.0
) -> List[str]:
    """
    RRF (Reciprocal Rank Fusion) 倒数排名融合算法
    无需调参，自动平衡两路结果的权重
    
    Args:
        es_results: ES检索结果，包含id和score
        qdrant_results: Qdrant检索结果，包含id和score
        k: 平滑常数，通常取值60
        es_weight: ES路权重
        qdrant_weight: Qdrant路权重
    
    Returns:
        融合后的文档ID列表，按融合分数降序排列
    """
    # 构建排名映射
    es_rank = {}
    for idx, item in enumerate(es_results):
        es_rank[item["id"]] = {
            "rank": idx + 1,
            "score": item.get("score", 0.0)
        }
    
    qdrant_rank = {}
    for idx, item in enumerate(qdrant_results):
        qdrant_rank[item["id"]] = {
            "rank": idx + 1,
            "score": item.get("score", 0.0)
        }
    
    # 合并所有文档ID
    all_ids = set(es_rank.keys()) | set(qdrant_rank.keys())
    
    # 计算RRF融合分数
    fusion_scores = {}
    for doc_id in all_ids:
        score = 0.0
        
        if doc_id in es_rank:
            score += es_weight / (es_rank[doc_id]["rank"] + k)
        
        if doc_id in qdrant_rank:
            score += qdrant_weight / (qdrant_rank[doc_id]["rank"] + k)
        
        fusion_scores[doc_id] = score
    
    # 按融合分数降序排序
    sorted_ids = sorted(fusion_scores.keys(), key=lambda x: fusion_scores[x], reverse=True)
    
    return sorted_ids, fusion_scores
```



#### 5.3.8 BGE Reranker精排

python

```
from transformers import AutoModelForSequenceClassification, AutoTokenizer

class BGEReranker:
    """BGE重排序器"""
    
    def __init__(self):
        self.model = AutoModelForSequenceClassification.from_pretrained(
            "BAAI/bge-reranker-v2-m3"
        )
        self.tokenizer = AutoTokenizer.from_pretrained("BAAI/bge-reranker-v2-m3")
    
    async def rerank(
        self,
        query: str,
        documents: List[Dict],
        top_k: int = 10
    ) -> List[Dict]:
        """
        对检索结果进行精排
        
        Args:
            query: 用户查询
            documents: 待排序文档列表，每个文档包含content字段
            top_k: 返回前k个结果
        
        Returns:
            精排后的文档列表
        """
        pairs = [(query, doc["content"]) for doc in documents]
        
        with torch.no_grad():
            inputs = self.tokenizer(
                pairs,
                padding=True,
                truncation=True,
                return_tensors="pt",
                max_length=512
            )
            scores = self.model(**inputs, return_dict=True).logits.view(-1).float()
        
        # 按分数降序排序
        for i, doc in enumerate(documents):
            doc["rerank_score"] = float(scores[i])
        
        sorted_docs = sorted(documents, key=lambda x: x["rerank_score"], reverse=True)
        
        return sorted_docs[:top_k]
```

class GraphRAGService:
    """知识图谱RAG服务"""
    
    async def query_graph(self, cypher: str) -> List[Dict]:
        """执行Cypher查询"""
        with self.neo4j_driver.session() as session:
            result = session.run(cypher)
            return [record.data() for record in result]
    
    async def find_patent_risks(self, product_specs: Dict) -> List[Dict]:
        """查询专利侵权风险"""
        query = """
        MATCH (f:Feature)
        WHERE f.name IN $features
        MATCH (p:Patent)-[:COVERS]->(f)
        WHERE p.jurisdiction IN $markets
        OPTIONAL MATCH (p)-[:OWNED_BY]->(c:Company)
        RETURN p.patent_id, p.title, p.legal_status, c.name as owner, f.name as feature
        """
        return await self.query_graph(query, {
            "features": list(product_specs.keys()),
            "markets": product_specs.get("target_markets", ["US"])
        })
    
    async def find_supply_chain_risks(self, supplier_id: str) -> Dict:
        """查询供应链风险"""
        query = """
        MATCH (s:Supplier {id: $supplier_id})
        OPTIONAL MATCH (s)-[:SUPPLIES]->(p:Product)
        OPTIONAL MATCH (p)-[:BELONGS_TO]->(c:Category)
        OPTIONAL MATCH (s)-[:LOCATED_IN]->(r:Region)
        OPTIONAL MATCH (r)<-[:AFFECTS]-(e:GeoPoliticalEvent {active: true})
        OPTIONAL MATCH (s)<-[:MENTIONS]-(n:News {sentiment: 'negative'})
        RETURN s, collect(DISTINCT p) as products, collect(DISTINCT c) as categories,
               collect(DISTINCT e) as geo_risks, count(n) as negative_news_count
        """
        return await self.query_graph(query, {"supplier_id": supplier_id})
    
    async def find_similar_success_cases(self, product_features: Dict) -> List[Dict]:
        """查找相似的成功案例"""
        query = """
        MATCH (p:Product {status: 'success'})
        WHERE p.category = $category
        MATCH (p)-[:HAS_FEATURE]->(f:Feature)
        WHERE f.name IN $features
        WITH p, count(f) as feature_match
        MATCH (p)-[:ACHIEVED]->(m:Metric)
        RETURN p.asin, p.title, feature_match, m.sales_volume, m.profit_margin, m.roi
        ORDER BY feature_match DESC, m.roi DESC
        LIMIT 10
        """
        return await self.query_graph(query, {
            "category": product_features.get("category"),
            "features": list(product_features.keys())
        })

    """多模态处理器"""
    
    async def process_image(self, image_url: str) -> Dict:
        """处理图片：生成向量和描述"""
        # 1. 下载图片
        image = await self.download_image(image_url)
        
        # 2. CLIP向量化
        image_vector = await self.clip_model.encode_image(image)
        
        # 3. 使用多模态LLM生成描述
        description = await self.vlm.describe_image(image)
        
        # 4. 提取视觉特征标签
        visual_tags = await self.extract_visual_tags(image)
        
        return {
            "vector": image_vector,
            "description": description,
            "tags": visual_tags,
            "source_url": image_url
        }
    
    async def process_video(self, video_url: str) -> Dict:
        """处理视频：提取关键帧和音频转录"""
        # 1. 下载视频
        video_path = await self.download_video(video_url)
        
        # 2. 提取关键帧（每5秒一帧）
        key_frames = await self.extract_key_frames(video_path, interval=5)
        
        # 3. 关键帧向量化
        frame_vectors = []
        frame_descriptions = []
        for frame in key_frames:
            vector = await self.clip_model.encode_image(frame)
            desc = await self.vlm.describe_image(frame)
            frame_vectors.append(vector)
            frame_descriptions.append(desc)
        
        # 4. 提取音频并转录
        audio_path = await self.extract_audio(video_path)
        transcript = await self.whisper_model.transcribe(audio_path)
        
        # 5. 转录文本向量化
        transcript_vector = await self.text_embedder.encode(transcript)
        
        return {
            "key_frame_vectors": frame_vectors,
            "frame_descriptions": frame_descriptions,
            "transcript": transcript,
            "transcript_vector": transcript_vector,
            "source_url": video_url
        }

class RAGService:
    """RAG检索服务 - 供Agent调用"""
    
    async def retrieve_knowledge(
        self,
        query: str,
        knowledge_type: str = "all",
        top_k: int = 10,
        filters: Dict = None
    ) -> List[Dict]:
        """
        通用知识检索
        
        Args:
            query: 查询文本
            knowledge_type: 知识类型 (all/product/review/doc/patent)
            top_k: 返回条数
            filters: 过滤条件
        """
        # 混合检索
        es_results = await self.es_retriever.search(query, knowledge_type, filters, top_k=50)
        qdrant_results = await self.qdrant_retriever.search(query, knowledge_type, filters, top_k=50)
        
        # RRF融合
        fused_ids, _ = rrf_fusion(es_results, qdrant_results)
        
        # 获取文档内容
        documents = await self._fetch_documents(fused_ids[:50])
        
        # Rerank精排
        reranked = await self.reranker.rerank(query, documents, top_k=top_k)
        
        return reranked
    
    async def retrieve_similar_cases(
        self,
        product_features: Dict,
        top_k: int = 5
    ) -> List[Dict]:
        """基于产品特征向量，检索历史上最相似的选品成功/失败案例"""
        feature_text = self._format_features(product_features)
        return await self.retrieve_knowledge(
            query=feature_text,
            knowledge_type="case_study",
            top_k=top_k,
            filters={"status": ["success", "failed"]}
        )
    
    async def retrieve_compliance_rules(
        self,
        product_category: str,
        target_market: str
    ) -> List[Dict]:
        """检索特定品类在目标市场的合规要求与法规"""
        query = f"{product_category} {target_market} 合规要求 认证标准"
        return await self.retrieve_knowledge(
            query=query,
            knowledge_type="doc",
            top_k=10,
            filters={"doc_type": "regulation", "market": target_market}
        )
    
    async def retrieve_competitor_analysis(
        self,
        asin: str
    ) -> Dict:
        """检索竞品分析知识"""
        # 先查图谱获取竞品关系
        competitors = await self.graph_service.find_competitors(asin)
        
        # 再检索竞品的评论和价格知识
        analysis = {}
        for comp_asin in competitors:
            reviews = await self.retrieve_knowledge(
                query=f"{comp_asin} 用户评价 优缺点",
                knowledge_type="review",
                top_k=20
            )
            analysis[comp_asin] = {
                "reviews": reviews,
                "pain_points": self._extract_pain_points(reviews)
            }
        
        return analysis

        )

        profit_analysis = await self._calculate_profit(
            pricing_strategy["recommended_price"],
            total_cost,
            market_analysis["market_size"]["sam"]
        )
        
        # 5. ROI预测
        roi_prediction = await self._predict_roi(
            profit_analysis,
            market_analysis["lifecycle_stage"],
            price_elasticity
        )
        
        # 6. 备货量建议
        inventory_plan = await self._generate_inventory_plan(
            market_analysis["market_size"]["sam"],
            roi_prediction["expected_sales"],
            state["budget_range"]
        )
        
        # 7. Go/No-Go决策
        go_decision = self._make_go_decision(
            profit_analysis["profit_margin"],
            roi_prediction["roi"],
            state["target_roi"]
        )
        
        return {
            "profit_analysis": {
                "cost_breakdown": {
                    "bom": bom_cost,
                    "logistics": logistics_cost,
                    "fba": fba_fees,
                    "advertising": ad_cost_estimate,
                    "total": total_cost
                },
                "pricing_strategy": pricing_strategy,
                "profit_margin": profit_analysis["profit_margin"],
                "net_profit": profit_analysis["net_profit"],
                "roi_prediction": roi_prediction,
                "inventory_plan": inventory_plan,
                "go_decision": go_decision
            }
        }
```

    """风险评估Agent"""
    
    async def execute(self, state: SelectionState) -> Dict:
        product_plan = state["product_plan"]
        profit_analysis = state["profit_analysis"]

            profit_analysis["inventory_plan"]["suppliers"]

            state["category"],
            state["target_market"],
            product_plan["recommended_specs"]

        market_risks = await self._assess_market_risk(
            state["market_analysis"]
        )
        risks.extend(market_risks)
        
        # 6. 图谱关系风险查询

            profit_analysis["inventory_plan"]["primary_supplier_id"]
        )
        risks.extend(graph_risks)
        
        # 7. 计算综合风险等级
        risk_level = self._calculate_risk_level(risks)
        
        return {
            "risk_assessment": {
                "risks": risks,
                "risk_level": risk_level,
                "risk_count": len(risks),
                "high_risk_count": len([r for r in risks if r["level"] == "high"]),
                "mitigation_summary": self._generate_mitigation_summary(risks)

        }

### 6.5 Human-in-the-loop人工干预设计





| 干预点             | 触发条件                   | 干预方式       | 干预后动作                   |
| :----------------- | :------------------------- | :------------- | :--------------------------- |
| **数据采集异常**   | 数据源采集失败率>30%       | 前端弹窗提醒   | 可选择重试/跳过/使用缓存     |
| **市场机会评分低** | opportunity_score < 50     | 推送至待审列表 | 人工确认是否继续分析         |
| **ROI不达标**      | predicted_roi < target_roi | 工作流暂停     | 人工调整定价或成本参数后继续 |
| **高风险等级**     | risk_level == "high"       | 强制人工审批   | 审批通过后方可进入采纳环节   |
| **采纳决策**       | Go/No-Go决策               | 运营人员确认   | 确认后触发ERP执行            |

```
class HumanInTheLoopService:
    """人工干预服务"""
    
    async def request_intervention(
        self,
        task_id: str,
        intervention_type: str,
        context: Dict,
        options: List[Dict]
    ) -> str:
        """请求人工干预"""
        intervention_id = generate_uuid()
        
        # 创建干预记录
        await self._create_intervention_record(intervention_id, task_id, intervention_type, context)
        
        # 推送通知
        await self.notification_service.send(
            user_id=context["owner_id"],
            title=f"选品任务需要您的决策 - {intervention_type}",
            content=self._format_intervention_content(context, options),
            actions=options
        )
        
        # 挂起工作流
        await self.workflow_manager.pause(task_id, intervention_id)
        
        return intervention_id
    
    async def submit_decision(
        self,
        intervention_id: str,
        decision: str,
        modified_params: Dict = None
    ) -> None:
        """提交人工决策"""
        # 更新干预记录
        await self._update_intervention_record(intervention_id, decision, modified_params)
        
        # 恢复工作流
        await self.workflow_manager.resume(intervention_id, decision, modified_params)
```

```
class LLMGateway:
    """LLM网关 - 智能路由"""
    
    async def route_and_generate(self, request: LLMRequest) -> LLMResponse:
        # 1. 检查租户配额
        if not await self._check_quota(request.tenant_id):
            raise QuotaExceededError("租户配额已用尽")
        
        # 2. 根据任务类型选择模型
        model_config = await self._select_model(request)
        
        # 3. 检查主模型健康状态
        if not await self._is_healthy(model_config.primary):
            model_config = await self._fallback_model(model_config)
        
        # 4. 检查队列长度
        if await self._queue_length(model_config.primary) > model_config.max_queue:
            model_config = await self._fallback_model(model_config)
        
        # 5. 执行推理
        start_time = time.time()
        try:
            response = await self._infer(model_config.primary, request)
            latency = time.time() - start_time
            
            # 6. 记录成本
            await self._record_cost(request.tenant_id, model_config.primary, response.tokens)
            
            return response
        except Exception as e:
            # 7. 失败降级
            logger.warning(f"模型 {model_config.primary} 推理失败: {e}")
            fallback_response = await self._infer(model_config.fallback, request)
            return fallback_response
    
    async def _select_model(self, request: LLMRequest) -> ModelConfig:
        """根据任务选择模型"""
        if request.task_type == "sensitive_filter":
            return ModelConfig(primary="phi-3-mini", fallback="rule_engine")
        elif request.task_type == "classification":
            return ModelConfig(primary="deepseek-v3", fallback="qwen2.5-7b")
        elif request.task_type == "complex_reasoning":
            return ModelConfig(primary="qwen2.5-72b", fallback="gpt-4")
        elif request.task_type == "multimodal":
            return ModelConfig(primary="qwen-vl-9b", fallback="text_fallback")
        elif request.task_type == "code_generation":
            return ModelConfig(primary="deepseek-coder", fallback="qwen2.5-7b")
        else:
            return ModelConfig(primary="qwen2.5-72b", fallback="deepseek-v3")

### B. 架构核心公式

> **AI选品系统 = （数据中台 + RAG知识系统） × Multi-Agent决策引擎 ÷ ERP闭环执行**

- **Agent** = 替代人脑决策（LangGraph + AutoGen + CrewAI + Dify）
- **RAG** = 沉淀企业经验，越用越强（Qdrant + ES + GraphRAG）
- **数据中台** = 核心竞争力，竞品无法复制（Flink + Iceberg + Feast）
- **ERP融合** = 真正商业价值，形成自进化飞轮（SCM + WMS + OMS + CRM + FMS）
- **外部数据** = 拓宽视野，发现蓝海（Amazon + TikTok + 1688 + GDELT + 爬虫）
