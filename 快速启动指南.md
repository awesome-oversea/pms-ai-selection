## 一、15 分钟快速了解项目

### 1.1 项目是什么？（3 分钟）

**项目名称**：跨境电商 AI 选品系统（PMS 增强版）

**系统定位**：

> 通过 AI 技术实现智能选品决策、市场趋势预测、竞品分析、供应链优化、利润最大化的企业级决策支持系统

**核心价值**：

- 选品效率提升 90%（2\~4 周/款 → 4 小时/款）
- 爆款命中率提升 35%（\~60% → ≥85%）
- 决策周期缩短 95%（周级 → 分钟级）
- 人工成本降低 60%（3\~5 人 → 0.5 人）

**技术栈**：

- 前端：Next.js 14 + Vue3 + ElementUI Plus + WebSocket + ECharts
- 后端：FastAPI + Python 3.11
- AI：Ollama + Qwen2.5-1.5B + Qwen3.5-2B（`qwen3.5:2b`）+ bge-reranker-base + Whisper tiny
- 数据：PostgreSQL + Redis + Kafka + Qdrant + OpenSearch + Iceberg
- 部署：K8s + Docker + Kong + Prometheus + Grafana

### 1.2 当前项目状态？（3 分钟）

**当前定位**：

> **企业级单体治理基线 + 多项样板级平台能力已完成**

**已完成的能力**：

- ✅ 企业级单体治理基线（多租户/RBAC/审计/Trace）
- ✅ Repository + Service 架构（8 个 Repository + 25+ 个 Service）
- ✅ Worker 化执行架构（SelectionTaskWorker + DataSyncConsumer）
- ✅ LLM Gateway（多模型路由 + 熔断器 + 四节点集群模拟）
- ✅ RAG 知识库（版本化 + 混合检索 + 评测）
- ✅ 数据同步（Kafka 事件发布 + 死信处理 + 幂等性）
- ✅ ERP 集成接口（7 大系统接口定义）
- ✅ 网关治理（Kong 配置拆分 + 路由分层）
- ✅ Agent 平台（拓扑构建 + 运行监控）
- ✅ 前端工程（Next.js 14 + 4 个页面）
- ✅ 测试体系（150+ 测试文件）

**核心差距**：

> **从样板级闭环到企业级落地终态的距离**

**10 个关键差距维度**：

1. 前端工作台不完整
2. 网关未真实部署
3. AI 中台未拆分
4. 检索平台未独立
5. 数据接入不完整
6. 数据湖未实现
7. ERP 集成不完整
8. Agent 平台缺可视化
9. 监控运维未部署

### 1.5 如何开始工作？（3 分钟）

**步骤 1：环境准备**

```bash
# 1. 克隆代码（如果还没有）
git clone [仓库地址]
cd fms

# 2. 安装依赖
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -e .

# 3. 启动本地依赖服务
python scripts/start_local_services.py

# 4. 启动后端容器
docker compose -f docker-compose.yml up -d --build --no-deps app

# 5. 启动前端
cd frontend
npm install
npm run dev
```

**步骤 2：领取任务**

- 查看 `企业级任务清单（基于完整差异分析）.md`
- 找到分配给你的任务

**步骤 3：开始开发**

- 阅读相关设计文档

- 提交代码

## 五、开发规范

### 5.1 代码规范

**Python 代码**：

- 遵循 PEP 8
- 使用 4 空格缩进
- UTF-8 编码
- 类型注解（mypy --strict）
- 文档字符串（Google 风格）

**前端代码**：

- 遵循 ESLint 配置
- 使用 TypeScript
- 组件化开发
- 响应式设计

### 5.2 提交规范

**提交信息格式**：

```
[Phase X] [任务编号] 提交信息

- 修改内容 1
- 修改内容 2
- ...

关联 Issue: #XXX
```

**示例**：

```
[Phase 1] [T1.1] 集成 ElementUI Plus

- 安装 ElementUI Plus
- 配置按需加载
- 测试基础组件

关联 Issue: #123
```

### 5.3 测试规范

**单元测试**：

- 每个函数/方法都要有单元测试
- 测试覆盖率≥80%
- 使用 pytest

**集成测试**：

- 每个 API 端点都要有集成测试
- 使用 pytest-asyncio
- 测试完整流程
